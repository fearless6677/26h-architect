面向对象架构演化——对象演化与消息演化

对象演化包括Add Object(AO)和Delete Object(DO)；消息演化包括AM/DM/SMO/OM/CMM五类操作

📖 核心解释

对象演化：AO在需要新功能或拆分功能时发生；DO在移除功能或合并对象时发生。消息演化5种操作：AM(增添消息)、DM(删除消息)、SMO(交换两条消息顺序)、OM(反转发送/接收对象)、CMM(改变消息发送或接收对象)。这些是架构动态行为层面的演化。

💻 伪代码

对象演化:
  AO - Add Object (新功能/拆分)
  DO - Delete Object (移除/合并)

消息演化:
  AM  - Add Message
  DM  - Delete Message
  SMO - Swap Message Order (交换顺序)
  OM  - Overturn Message (反转发送/接收)
  CMM - Change Message Module (改变发送/接收对象)

📌 实际例子

  • 需要新增订单处理功能 → AO(Add Object)
  • 合并两个相似服务对象 → DO(Delete Object)
  • 先验证权限再登录改为先登录再验证 → SMO
  • 消息原来A→B改为B→A → OM
  • 消息接收方从服务A改为服务B → CMM

✅ 优点

  • 提供细粒度的演化操作
  • 可精确描述架构行为变更

❌ 缺点

  • 操作种类多，需仔细分析影响范围
  • 组合使用复杂度高
