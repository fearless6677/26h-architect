编译优化：减少冗余指令，降低CPU执行时间； 软硬件协同：根据任务动态调配硬件资源（如休眠外设）； 算法优化：用中断代替轮询、查表代替复杂运算，减少CPU运行时间。

// 低功耗设计技术
function reducePower(system) {
    compileOptimize(system);   // 编译优化：精简指令
    hwSwCoDesign(system);      // 软硬件协同：按需调配
    algorithmOptimize(system); // 算法优化：中断代轮询
}
核心：嵌入式软件六大特点围绕资源受限和确定性需求展开—— 可剪裁去掉冗余、可配置灵活适配、强实时保证响应、安全性分析故障(FMECA)、可靠性用容错/余度/鲁棒性、高确定性靠静态分配和状态机。 低功耗的三条路：编译优化、软硬件协同、算法优化。
✂️ 裁剪文件系统模块减小ROM ⏱️ 查表法代替复杂计算满足实时 ⚡ 中断代替轮询减少CPU时间 🛡️ FMECA = 故障模式影响及危害性分析

✅ 优点

  • 针对性强、资源利用率高
  • 裁剪后可在极小ROM/RAM上运行
  • 实时性和确定性满足关键任务需求

❌ 挑战

  • 开发复杂度高，需深入硬件细节
  • 调试手段有限（资源受限）
  • 安全性/可靠性设计增加开发成本

📝 考试关键词

可剪裁 可配置 FMECA 容错 余度技术 鲁棒性 编译优化 软硬件协同设计 算法优化
