数据库交互技术演进时间线

第1阶段
库函数 (如 Oracle OCI)

最底层的API调用，直接调用数据库提供的函数库。
✅ 性能最好 ❌ 强依赖特定DB，难移植

第2阶段
嵌入式 SQL

将SQL语句直接写入高级语言源代码中。
✅ 代码与SQL混合 ❌ 需要预编译器处理

第3阶段
通用接口 (ODBC / JDBC)

实现与DBMS无关的统一访问标准。
✅ 跨数据库 ✅ 统一接口 ❌ 仍需写SQL

第4阶段
ORM (对象关系映射)

将面向对象语言中的对象与数据库表映射。
✅ 开发效率高 ✅ 面向对象操作数据库 ❌ 复杂SQL不够灵活

// ====== 1. 库函数 (OCI) —— 强依赖特定DB ======
OCIEnv env;
OCIEnvCreate(&env, OCI_DEFAULT);      // 初始化Oracle环境
OCIStmtExecute(svc, stmt, err);       // 执行SQL —— 只能用于Oracle!

// ====== 2. 嵌入式SQL —— SQL混在代码中 ======
EXEC SQL SELECT name INTO :name_var FROM users WHERE id = :id;  // 需要预编译

// ====== 3. ODBC/JDBC —— 统一接口，跨数据库 ======
// ODBC (Windows): Access/MySQL/SQL Server 统一访问
Connection conn = DriverManager.getConnection("jdbc:mysql://localhost/db");
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM users");

// ====== 4. ORM —— 面向对象操作数据库 ======
// Hibernate (全自动): 写对象自动变SQL
User user = new User("张三", 25);
session.save(user);  // 自动生成 INSERT INTO users ...

// Mybatis (半自动): 手写SQL但灵活
User user = userMapper.selectById(1);  // XML中定义SQL语句
🔌 ODBC 实例

Windows下统一访问 Access / MySQL / SQL Server 的接口。只需换驱动，代码不用改。

☕ JDBC 实例

Java程序连接 MySQL 的标准方式。DriverManager.getConnection("jdbc:mysql://...")

🧠 Hibernate (全自动 ORM)

写Java对象自动映射为SQL。开发者只需操作对象，不需要写SQL语句。

⚡ Mybatis (半自动 ORM)

需要手写SQL但灵活度高。适合复杂查询场景，SQL写在XML文件中与Java代码分离。

核心：应用程序与数据库的交互方式不断演进： 库函数(如Oracle OCI)是最底层接口，强依赖特定数据库； 嵌入式SQL将SQL语句写入高级语言源代码中； 通用数据接口标准(ODBC/JDBC)实现了与DBMS无关的统一访问； ORM(对象关系映射)将面向对象语言中的对象与数据库表映射，典型框架有 Hibernate(全自动)、Mybatis(半自动)和JPA(Java自带)。 ORM让开发者以操作对象的方式来操作数据库。
🔌 ODBC: Windows统一DB接口 ☕ JDBC: Java连接DB标准 🧠 Hibernate: 全自动ORM ⚡ Mybatis: 半自动ORM 📝 嵌入式SQL: SQL嵌入代码 🗄️ OCI: Oracle专用API

✅ 优点

  • ORM开发效率高——操作对象=操作数据库
  • 通用接口(ODBC/JDBC)可移植性好——换DB不改代码
  • ORM降低学习成本——不用精通SQL也能开发

❌ 缺点

  • ORM性能有损耗——多了一层转换
  • 复杂SQL不够灵活——ORM难以处理复杂查询
  • 库函数移植性差——绑定特定数据库

📝 考试关键词

ODBC JDBC ORM Hibernate Mybatis JPA 嵌入式SQL OCI

🔗 关联知识

  • NoSQL与Redis = 另一种数据库交互范式（非关系型、键值存储）
  • JPA = Java自带的ORM规范，Hibernate是其实现之一
  • 数据库中间件 = 介于应用和数据库之间的通用访问层
  • 连接池 = ODBC/JDBC中复用数据库连接的技术
