// 以太网帧格式约束 (IEEE 802.3)
ETHERNET_MIN_FRAME = 64 字节
ETHERNET_MAX_FRAME = 1518 字节

// 最小帧长目的：避免冲突
// 最小帧长依据：网络中检测冲突的最长时间

function validateFrame(frame) {
    if (frame.length < 64)
        return discard("冲突碎片")  // 丢弃！
    if (frame.length > 1518)
        return discard("超长帧")
    return accept(frame)
}

// WLAN标准 (IEEE 802.11)
802.11a/b/g → 早期标准
802.11n    → 200 Mb/s
802.11ac   → 1 Gb/s
核心：以太网(IEEE 802.3)规定帧长范围 64 ~ 1518 字节。 设置最小帧长是为了避免冲突，依据是网络中检测冲突的最长时间。 WLAN 采用 IEEE 802.11 标准族，802.11n 可达 200 Mb/s，802.11ac 可达 1 Gb/s。
🔌 以太网 IEEE 802.3 📡 WLAN IEEE 802.11 📐 最小帧长 64B 📏 最大帧长 1518B ⚡ CSMA/CD 冲突检测

✅ 优点

  • 最小帧长机制有效避免冲突检测问题
  • 帧长标准化保证 interoperability

❌ 限制

  • 最小帧长限制了短距离网络效率
  • 1518B 最大帧可能导致大数据传输分片
  • WLAN 不使用 CSMA/CD，机制不同于以太网

📝 考试关键词（必考）
