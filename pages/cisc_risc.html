CISC与RISC指令集对比

CISC指令复杂、功能强（x86代表）；RISC指令精简、采用Load/Store架构（ARM代表），国产CPU多用RISC。

CISC（x86）

① 取指 →
MOV [mem], R1
② 译码+执行 →
ALU 运算
③ 写回 →
结果写回 [mem]
✅ 1条指令搞定！

RISC（ARM）

① 载入 →
LOAD R1, [mem]
② 运算 →
ADD R2, R1, R3
③ 写回 →
STORE R2, [mem]
✅ 3条指令（流水线高效）
// CISC：单条指令功能强，可完成复杂操作
MOV [mem], R1   ; 可直接对内存操作，指令长度可变

// RISC：指令精简，每条指令功能简单，固定长度
LOAD  R1, [mem]  ; 必须先将数据载入寄存器
ADD   R2, R1, R3 ; 运算只能对寄存器操作
STORE R2, [mem]  ; 结果再写回内存
对比维度CISC（复杂指令集）RISC（精简指令集）
指令长度可变长度固定长度
指令数量多（数百~数千条）少（百余条）
寻址方式复杂多样简单（Load/Store）
内存操作可直接对内存运算只能Load/Store，运算在寄存器
流水线效率较低（指令复杂不均）高（指令均匀，适合流水线）
功耗较高较低
代表架构Intel/AMD x86ARM、MIPS、RISC-V
国产代表—龙芯（MIPS）、飞腾（ARM）、申威
核心思想：CISC 追求「让硬件做更多事」，单条指令功能强大但译码复杂； RISC 追求「让指令尽量简单」，采用 Load/Store 架构，运算只在寄存器间进行，天然适合流水线并行。 现代处理器实际上已经融合——x86 内部会将复杂指令微码拆分为类似 RISC 的微操作执行。
💻 Intel/AMD x86 → CISC代表 📱 ARM Cortex → RISC代表 🇨🇳 龙芯（MIPS）/ 飞腾（ARM）/ 申威 🔓 RISC-V → 开源指令集

✅ 优点

  • CISC：单条指令功能强，代码密度高
  • RISC：流水线效率高，功耗低，设计简单

❌ 缺点

  • CISC：指令复杂，译码器设计困难，功耗高
  • RISC：完成同等功能需要更多条指令，代码密度低
