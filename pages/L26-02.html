论文六段式框架与字数要求

摘要(290-320字) → 项目背景(5W2H) → 过渡 → 理论(400-600字) → 实践(1000-1200字) → 结尾(300-400字)

💻 写作模板
📐 5W2H详解
①
摘 要 项目背景 → 技术简介 → 效果简述（不分段） 290 ~ 320字
②
项目背景 采用 5W2H 模式展开（Why / When / Where / Who / What / How much / How） 400 ~ 500字
③
过 渡 承上启下，自然引出下文理论与实践 ≈ 100字
④
理 论 紧扣题干第二小问，论述基本概念和原理 400 ~ 600字
⑤
实 践 ⭐ 最重要部分！采用 Why + How 结构，提炼小标题 1000 ~ 1200字
⑥
结 尾 呼应论点、项目效果、个人收获与展望 300 ~ 400字

// 正文总字数：2200 ~ 2800字

一、摘要 // 290~320字，不分段
   项目背景介绍 → 项目技术简介 → 项目效果简述
   // 一段写完，不另起段落

二、项目背景 // 400~500字
   Why    — 项目由来，为什么要做
   When   — 项目起止时间
   Where  — 项目所在地 / 单位
   Who    — 甲方乙方、我的角色
   What   — 建设内容概述
   How much — 规模、预算
   How    — 采用的技术方法
   // 💡 这段可复用于不同题目！

三、过渡 // ≈100字
   承上启下，引出理论与实践内容

四、理论部分 // 400~600字
   紧扣题干第二小问
   → 基本概念定义
   → 核心原理阐述
   → 技术选型依据

五、实践部分 // 1000~1200字，最重要！
   Why  — 为什么要用这个技术/方法
   How  — 具体怎么做的（提炼小标题）
   ├── 小标题1：具体做法 + 遇到的问题 + 解决方案
   ├── 小标题2：具体做法 + 遇到的问题 + 解决方案
   └── 小标题3：具体做法 + 遇到的问题 + 解决方案

六、结尾 // 300~400字
   → 呼应论点（回应题目要求）
   → 项目效果（数据说话）
   → 个人收获与不足
   → 未来展望

核心：论文由六部分组成，其中实践部分(1000-1200字)最重要，占正文近一半篇幅，采用 Why+How 结构并提炼小标题。 正文总字数控制在 2200~2800字。

📐 5W2H 速记

Why(由来) / When(时间) / Where(地点) / Who(甲乙方) / What(建设内容) / How much(规模预算) / How(技术方法)

📖 摘要290-320字 📖 正文2200-2800字 📐 5W2H框架 🔗 过渡承上启下 📚 理论部分紧扣题干 ⭐ 实践部分Why+How

✅ 优点

  • 项目背景可复用于不同题目
  • 结构清晰便于阅卷
  • 字数分配明确，不易超标或不足
  • 实践部分占主体，突出技术深度
