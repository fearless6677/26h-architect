// 软件配置管理(SCM)核心
SCM = 版本控制 + 变更控制

版本控制: 追踪文件变更、支持并行开发
变更控制: 确保变更有序进行

// 软件质量保证(SQA)任务

3. 处理不符合问题

// 质量认证

// WBS工作分解结构
项目 → 任务 → 工作 → 活动(分到个人)
以可交付成果为导向，处于计划过程中心
核心：软件配置管理(SCM)标识、组织和控制修改，核心是版本控制(追踪文件变更、并行开发)和变更控制(确保变更有序)。目的是使错误最小并提高生产效率。

SQA：使软件过程对管理人员可见，主要任务：审计与评审、报告、处理不符合问题。质量认证主要采用ISO 9001和CMM。

WBS：以可交付成果为导向，将项目逐层分解到个人活动，处于计划过程中心。
🖥️ Git/SVN版本控制 🔧 CCB变更评审 📋 SQA过程审计 📊 WBS逐层分解

✅ 优点

  • 版本控制防止代码混乱
  • 变更控制防止随意修改
  • WBS使项目计划清晰可控
  • SQA保证过程可见可管理

❌ 缺点

  • 配置管理需要额外工具和流程开销
  • SQA审计可能影响开发进度
