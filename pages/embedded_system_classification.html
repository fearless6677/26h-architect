嵌入式系统的分类（实时与安全攸关）

嵌入式系统分实时/非实时，实时又分强实时（超时=灾难）和弱实时（偶尔超时没关系），还有安全攸关（坏了会死人）之分。

// 嵌入式系统分类树
嵌入式系统 {
  按实时性: {
    强实时(Hard): 超时=灾难
    弱实时(Weak): 偶尔超时可接受
    非实时: 无严格时间要求
  }
  按安全性: {
    安全攸关: 失效→伤亡/损失
    非安全攸关: 失效无严重后果
  }
}
核心：嵌入式系统按实时性分为实时系统和非实时系统。实时系统又分强实时（Hard Real-Time，必须在规定时间内完成，超时即灾难）和弱实时（Weak Real-Time，偶尔超时可接受）。按安全性分为安全攸关系统（失效导致人员伤亡或财产损失）和非安全攸关系统。
强实时系统：飞机飞控计算机，毫秒级响应，超时=坠机 弱实时系统：视频直播播放器，偶尔卡顿但不致命 安全攸关系统：核电站控制系统的嵌入式监控软件

✅ 优点

  • 分类明确，便于选择合适的设计策略
  • 安全攸关分类有助于确定可靠性等级

❌ 缺点

  • 强实时系统设计和验证成本高
  • 安全攸关系统需要严格的标准认证流程
