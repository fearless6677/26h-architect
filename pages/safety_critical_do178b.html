安全攸关软件与DO-178B标准

安全攸关软件=坏了会出大事的软件，DO-178B标准把它分A(灾难)到E(无影响)五个等级，等级越高验证越严格。

💻 伪代码
A级 · 灾难级 — 失效=机毁人亡
B级 · 危害级 — 失效=多人伤亡
C级 · 严重级 — 失效=少数人伤亡
D级 · 不严重级 — 失效=轻微伤害
E级 · 没有影响级

等级越高 → 验证越严格 → 成本越高

// DO-178B 软件安全等级
等级 = {
  A: 灾难级 (失效=机毁人亡),
  B: 危害级 (失效=多人伤亡),
  C: 严重级 (失效=少数人伤亡),
  D: 不严重级 (失效=轻微伤害),
  E: 没有影响级
}
// 软件生命周期三大过程
生命周期 = [软件计划, 软件开发(需求→设计→编码→集成), 软件综合(验证→配置管理→质量保证→审定)]
核心：安全攸关软件指其不正确功能或失效会导致不可接受风险的软件。DO-178B是机载软件安全性标准，将软件生命周期分为三大过程：软件计划过程、软件开发过程（需求→设计→编码→集成）、软件综合过程（验证→配置管理→质量保证→审定联络）。按重要程度分A~E五级：A灾难级、B危害级、C严重级、D不严重级、E没有影响级。
DO-178B A级：飞机自动着陆系统，失效=机毁人亡，需最严格验证 DO-178B C级：飞机客舱娱乐系统，失效不导致伤亡，要求较低 汽车ESP电子稳定系统：安全攸关软件，失效可能导致车祸

✅ 优点

  • 分级管理，不同等级匹配不同开发成本
  • 生命周期标准化，提高软件质量和可追溯性
  • 为适航审定提供客观依据

❌ 缺点

  • 认证流程复杂、周期长、成本高
  • 过度设计可能浪费资源
