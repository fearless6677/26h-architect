// UML 2.0 核心要素
// 4种事物: 结构(类/接口/用例) + 行为(交互/状态机) + 分组(包) + 注释
// 4种关系: 依赖(uses-a) → 关联(has-a) → 泛化(is-a) → 实现(realize)
//   聚合: 整体-部分，部分可属于多个整体（生命周期独立）
//   组合: 整体-部分，部分只属于一个整体（生命周期相同）
// 14种图: 类图/用例图/序列图/活动图/状态图/部署图...
// 5种视图: 用例视图(中心) / 逻辑视图 / 进程视图 / 实现视图 / 部署视图
核心：UML是统一建模语言，用14种图和4种关系来可视化描述软件系统的结构和行为。
电商系统中，'下单'用例包含'检查库存'子用例（include关系），'使用优惠券'是'下单'的扩展用例（extend关系） 汽车类与发动机类是组合关系（生命周期相同），汽车与轮胎是聚合关系（轮胎可换到其他车） 学生类继承自用户类是泛化关系(is-a)，订单类使用支付接口是实现关系

✅ 优点

  • 提供标准化的建模语言，使开发人员和用户有统一的沟通工具
  • 14种图覆盖软件生命周期的各个阶段，从需求到部署全面建模
  • 5种视图从不同视角描述系统，用例视图居于中心地位，以用户需求驱动

❌ 缺点

  • UML图种类多达14种，学习和掌握成本较高
  • 过于复杂的UML图难以维护，容易与实际代码脱节
  • 不同建模者对同一系统的UML图可能有不同理解，存在歧义
