决策树计算规则： 从最右端叶子开始，向左逐层回推。 机会节点的EMV = 各结果收益 × 对应概率之和。 决策节点选EMV最大（收益型）或最小（成本型）的方案。 
// ── EMV 期望货币值法 ──
function EMV(方案) {
    sum = 0
    for each (收益, 概率) in 方案.结果
        sum += 收益 × 概率       // 核心公式
    return sum
}

// ── 决策树：从右向左逐层计算 ──
function 决策树(node) {
    if node 是叶子   // ← 从最右端开始
        return node.收益
    if node 是机会节点
        return Σ 决策树(child) × child.概率
    if node 是决策节点
        return max(决策树(child₁), 决策树(child₂), ...)  // 选最优
}

// ── 后悔值准则 ──
function 后悔值(状态s, 方案a) {
    return max(同状态下所有方案收益) − 方案a在s下的收益
}
// 选各方案最大后悔值中的 → 最小值（最小最大后悔值）

后悔值矩阵示例：

方案 \ 状态S₁（好）S₂（中）S₃（差）最大后悔值
A 0 120 280 280
B 60 0 140 200 ✓
C 100 80 170 350
D 200 160 300 500

后悔值 = 同状态下最高收益 − 该方案收益。 各方案最大后悔值分别为 280 / 200 / 350 / 500， 取最小200 → 选方案B。

核心：决策分析从若干方案中选最优。 风险型（概率已知）用EMV/决策树； 不确定型（概率未知）用5种准则：
🔥 乐观准则

最大最大（大中取大）
选各方案最大收益中的最大

🧊 悲观准则

最大最小（小中取大）
选各方案最小收益中的最大

⚖️ 折中准则

乐观系数α加权
HV = α×最大 + (1-α)×最小

🎲 等可能性

假设各状态等概率
直接算EMV比较

😤 后悔值准则

最小最大后悔值
各方案最大后悔值中选最小

📦 货运决策（水路vs陆路） 🏭 建厂决策树（大厂vs小厂） 📊 不确定型后悔值矩阵

✅ 优点

  • EMV法量化客观，可比较
  • 决策树直观展示多阶段决策过程
  • 不确定型决策多种准则覆盖不同风险偏好

❌ 缺点

  • 需要已知各状态的概率，实际中概率估计可能不准确
  • 多阶段决策树计算量大，从右向左逐层计算容易出错
