⚡ 性能
正常负载下，查询请求响应时间 ≤ 2s 敏感点
高并发峰值（1000 QPS）时系统不崩溃 风险点
🔒 安全性
数据传输采用AES-256加密 权衡点
未授权访问检测与告警 ≤ 5s
🔧 可修改性
新增业务模块不影响现有功能 权衡点
配置变更无需重启系统
🛡️ 可用性
系统年可用率 ≥ 99.99%
故障恢复时间（RTO）≤ 30min 风险点
💡 三类关键点说明：
风险点 = 可能导致质量属性不达标的架构决策
敏感点 = 对某个质量属性有显著影响的架构决策
权衡点 = 同时影响多个质量属性的架构决策（改善一个可能恶化另一个） 
// ========== ATAM论文写作模板 ==========

一、项目背景与架构描述
  介绍项目背景 → 说明系统架构风格 → 画出架构图

二、评估方法选择
  选择理由：ATAM是基于场景的架构评估方法
  评估目标：关注性能、可用性、安全性、可修改性等质量属性

三、评估过程（四阶段）
  /* 阶段1：描述和介绍 */
  → 介绍ATAM方法、利益相关者、架构、质量需求

  /* 阶段2：调查和分析 ⭐核心 */
  → 识别风险点（可能导致问题的决策）
  → 识别敏感点（显著影响某质量属性的决策）
  → 识别权衡点（同时影响多个质量属性的决策）
  → 生成质量效用树

  /* 阶段3：测试 */
  → 对场景按重要性/难度排序
  → 针对高风险场景提出应对方案

  /* 阶段4：报告 */
  → 汇总评估结果，输出评估报告文档

四、评估结论与改进建议
  总结风险 → 给出架构改进建议 → 展望未来优化方向
核心：ATAM（架构权衡分析法）是基于场景的架构评估方法。 论文写作要体现完整的四阶段过程，重点在第2阶段——识别风险点、敏感点、权衡点，并生成质量效用树。
📝 敏感点示例：查询请求处理时间影响系统集群方式和处理过程设计 ⚖️ 权衡点示例：更改加密级别同时影响安全性和性能

✅ 范文优点

  • 范文涵盖架构评估核心知识点

⚠️ 注意事项

  • 需结合实际项目场景来写，避免空泛
  • 效用树要具体量化，不能只写概念

✍️ 论文写作技巧

  • 1开头先交代项目背景，说明为什么选择ATAM方法
  • 2质量效用树必须具体化——每个叶子节点要有可测量的场景描述
  • 3风险点/敏感点/权衡点要结合架构决策来说明，不能脱离架构谈评估
  • 4权衡点要体现"鱼与熊掌不可兼得"的矛盾关系（如加密级别↑ → 安全性↑ 但性能↓）
  • 5结尾要有改进建议，展示评估的实际价值
