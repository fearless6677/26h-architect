EmbeddedSystem = {
  处理器: MPU / MCU / DSP / SoC,    // 核心：分民用/工业/军用三档
  支撑硬件: [存储器, 定时器, 总线],   // 外围硬件支撑
  操作系统: RTOS,                   // 实时性、可裁剪、安全性
  支撑软件: 公共服务库,              // 以库方式运行在 OS 之上
  应用软件: 专用功能                 // 完成特定目标（如温控/导航）
}

// 核心特点
特点 = {
  中心: 以特定应用为中心,
  硬件: 可裁剪 + 低功耗 + 体积小,
  软件: 代码固化ROM + 资源受限,
  本质: 专用计算机系统,
  工具: 需要专门开发工具（交叉编译+仿真器）
}
处理器温度等级
🏠
民用级
0 ~ 70°C
🏭
工业级
-40 ~ 85°C
🎖️
军用级
-55 ~ 150°C

温度等级决定了处理器的可靠性和成本。 军用级 > 工业级 > 民用级（价格和可靠性依次递减）。

核心：嵌入式系统由 12345 五部分组成：
① 嵌入式处理器 — 核心，分民用级(0~70°C)、工业级(-40~85°C)、军用级(-55~150°C)三档；
② 支撑硬件 — 存储器、定时器、总线等；
③ 嵌入式操作系统 — 具备实时性、可裁剪性和安全性；
④ 支撑软件 — 以库方式运行在 OS 之上的公共服务；
⑤ 应用软件 — 完成专用目标，是系统存在的意义。
🏠 智能家居控制器：MCU+传感器+RTOS+APP 🚗 汽车ECU：工业级处理器+CAN总线+安全攸关软件 🏭 工业PLC：军用级处理器+多种I/O+实时OS

✅ 优点

  • 专用性强，资源高效利用
  • 体积小、功耗低、性价比高
  • 实时性好，响应迅速可靠
  • 软硬一体，整体优化空间大

❌ 局限性

  • 资源受限（内存/存储/算力有限）
  • 需要专门开发工具（交叉编译+仿真器）
  • 代码固化 ROM，升级维护困难
  • 安全性要求高，开发门槛高
