系统规划步骤

系统规划是信息系统生命周期的第一阶段，包括初步调查→确定目标→分析子系统→拟定方案→可行性研究→制订建设方案六个步骤

// 系统规划六步法
// Step1: 对现有系统进行初步调查
// Step2: 分析和确定系统目标
// Step3: 分析子系统的组成和基本功能
// Step4: 拟定系统的实施方案
// Step5: 进行可行性研究，编写可行性研究报告，召开可行性论证会
// Step6: 制订系统建设方案

function 系统规划(企业环境) {
    const 现状 = 初步调查(企业环境);         // Step1
    const 目标 = 确定系统目标(现状);           // Step2
    const 子系统 = 分析子系统(目标);            // Step3
    const 方案候选 = 拟定实施方案(子系统);      // Step4
    const 可行性 = 可行性研究(方案候选);          // Step5
    const 建设方案 = 制订建设方案(可行性);      // Step6
    return 编写系统设计任务书(建设方案);        // 产出物
}
核心：系统规划是信息系统生命周期的第一个阶段，其任务是对企业的环境、目标及现有系统的状况进行初步调查，研究建设新系统的必要性和可能性。根据需要与可能给出拟建系统的备选方案，进行可行性分析，写出可行性研究报告。报告审议通过后编写系统设计任务书。

✅ 优点

  • 为后续系统建设提供明确方向和依据
  • 通过可行性分析降低项目失败风险

❌ 局限

  • 规划阶段投入有限，难以全面覆盖所有可能性

📝 考试关键词
