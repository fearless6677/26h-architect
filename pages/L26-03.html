论文项目选择策略（L26-03）

项目背景首选亲历项目，近三年为宜，地点和甲方需脱敏处理。

🎬 优先级动画
🔒 脱敏演示
💻 选择伪代码
① 亲历项目 参与过、经历过
↓
② 熟悉项目 未参与但能理解
↓
③ 文档项目 通过文档理解

项目时间范围：近三年  |  工期：半年至一年

论文中必须脱敏的信息：

北京市朝阳区 → 某市
某某科技有限公司 → 我司 / 我单位
甲方：XX银行 → 某金融机构 / 甲方
承建方：华为 → 我单位 / 我公司

💡 地点写"某省/某市"而非实际城市名，乙方称"我司/我单位/我厂/我公司"

// 论文项目选择 —— 优先级决策
function selectProject(projects) {
    // 硬性条件：近三年 + 工期半年至一年
    let candidates = projects.filter(p =>
        p.year >= 近三年 &&
        p.duration >= 半年 &&
        p.duration <= 一年
    );

    // 优先级排序
    return candidates.sort((a, b) => {
        if (a.is亲历) return -1;   // ① 亲历优先
        if (a.is熟悉) return 1;    // ② 熟悉次之
        return 2;                    // ③ 文档兜底
    });
}

// 选定后脱敏处理
function anonymize(project) {
    project.地点 = "某省/某市";         // 隐去真实地名
    project.甲方 = "某机构";             // 隐去甲方真名
    project.乙方 = "我司/我单位";         // 乙方统称
}
核心原则：越熟悉的项目，论文材料越充实，论述越能充分。 选项目就像选答题——选自己最会的，才能写出有血有肉的内容。
📅 近三年项目 ⏱️ 工期半年至一年 🔒 地点脱敏 → 某省/某市 🏢 甲方脱敏 → 某机构 🏭 乙方 → 我司/我单位

✅ 优点

  • 越熟悉越容易写，素材丰富
  • 项目背景可在多篇论文中复用
  • 亲历项目细节真实，说服力强

❌ 注意事项

  • 地点、甲方名称必须脱敏处理
  • 不熟悉的项目论述会空洞无力
  • 项目过旧或工期过短不适合

📝 考试关键词

近三年项目 工期半年至一年 项目脱敏 亲历优先

💡 实战建议

提前准备 1~2 个"万能项目"，把项目背景、技术架构、团队分工等内容整理成模板。 考试时根据不同论文题目，从同一项目中截取不同侧面来论述，省时又高效。
