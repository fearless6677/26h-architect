分布式数据库与分布透明性

分布式数据库就是把数据分散存放在多台服务器上，像一个团队分工合作，既快又不会因一个人请假而停工。

🏆 分片透明（最高层）
用户完全不知道数据怎么分片，只操作全局表
📍 位置透明（中间层）
知道分片了，但不用知道数据存在哪个节点
🔧 局部数据模型透明（最低层）
知道位置了，但不用知道底层用了什么数据模型
透明性由高到低：分片透明 → 位置透明 → 局部数据模型透明
水平分表：订单表按ID范围切分

// ═══ 分布透明性层级（由高到低）═══
// 分片透明(最高): 用户只管操作全局表
SELECT * FROM orders  -- 不知道数据被分成3片

// ═══ 主从复制（binlog同步）═══
Master ──binlog──→ Slave1  // 全同步：最安全，最慢
Master ──binlog──→ Slave2  // 半同步：折中方案
Master ──binlog──→ Slave3  // 异步复制：最快，可能丢数据

// ═══ 读写分离 ═══
write(data) → Master       // 写操作只走主库
read(key)  → Slave1|Slave2  // 读操作分散到从库

// ═══ 分表策略 ═══
// 水平分表：按行切分（同列，不同行）
split(orders, "按ID范围") → table_1, table_2
// 垂直分表：按列切分（同行，不同列）
split(trade, "热列+冷列") → hot_table, cold_table

// ═══ 分库策略 ═══
deploy("用户", DB_A)  // 不同业务放不同实例
deploy("商品", DB_B)
deploy("订单", DB_C)
核心：分布式数据库将数据分布在不同物理节点上， 具有共享性、自治性、可用性、分布性四大特点。 透明性分三层——分片透明（用户不用关心数据怎么分片）、 位置透明（知道分片但不用知道存在哪）、 局部数据模型透明（知道位置但不用知道用了什么数据模型）。 优化技术包括主从复制(binlog同步)、读写分离、分表(水平/垂直)、分库。
🔄 主从复制(binlog同步) 📖 读写分离(主写从读) 📊 水平分表(按行) 📐 垂直分表(按列) 🏢 分库(业务隔离)
🏆 分片透明

用户只操作全局表，不知道数据被分成3片存在不同服务器上

📊 水平分表

订单表按ID范围分成表1(1-999999)和表2(1000000-9999999)

📐 垂直分表

交易表拆成热表(ID/状态/金额)和冷表(备注/物流信息)

🏢 分库

电商网站把用户、商品、订单分别放到三台不同的数据库服务器

✅ 优点

  • 高可用性（单点故障不影响整体）
  • 水平扩展能力强
  • 就近访问提高性能

❌ 缺点

  • 一致性保证困难
  • 分布式事务复杂
  • 运维成本高

📝 考试关键词
