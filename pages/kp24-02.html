最大流量问题（KP24-02）

运输网络中从源点到汇点的最大运输能力，逐路径求最小值并扣除剩余流量

// 最大流量 —— 逐路径找最小容量，扣除已用流量
function maxFlow(network, source, sink) {
    totalFlow = 0

    while (findPath(network, source, sink)) {   // 重复找可达路径
        path = findPath(network, source, sink)
        pathFlow = min(path.各边剩余容量)     // 路径流量 = 瓶颈

        for (edge in path) {
            edge.剩余容量 -= pathFlow           // 扣除已用流量
            if (edge.剩余容量 == 0)
                edge.断开()                     // 容量耗尽 → 边断开
        }
        totalFlow += pathFlow                   // 累加到总流量
    }

    return totalFlow                            // 无路径可达 → 结束
}

// 示例：节点①→⑥
// 路径①③⑤⑥ = min(…) = 10
// 路径①②⑤⑥ = min(…) = 6
// 路径①④⑥   = min(…) = 5
// 路径①④③⑤⑥ = 1, 路径①④②⑤⑥ = 1
// 总流量 = 10+6+5+1+1 = 23 万吨/小时
核心：在运输网络中找从源点到汇点的最大运输能力。 方法：逐条找出可行路径，每条路径流量 = 该路径各段容量的最小值（瓶颈决定流量）。 算完一条后扣除已用流量（剩余为0的边断开），重复直到无路径可达。总最大流量 = 各路径流量之和。本质是特殊的线性规划问题。
🚰 管道/运输网络 📊 线性规划特例 🔁 逐路径迭代求解 ⛓️ 瓶颈 = 路径最小容量

✅ 优点

  • 思路清晰，逐步迭代易于理解
  • 适用于管道、运输等实际网络问题

❌ 缺点

  • 需要手动逐步迭代，计算过程繁琐
  • 路径选择和流量扣除容易出错
