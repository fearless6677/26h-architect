逆向工程与软件重用

重用是'拿来主义'复用已有代码；逆向工程是从代码反推出设计文档。

📖 核心解释

软件重用是使用已有产品开发新系统的过程，分水平式重用(不同领域，如标准函数库)和垂直式重用(共性领域，如区块链)。逆向工程通过分析程序寻求更高级抽象(如文档)，是不同抽象层级的溯源行为，分为实现级→结构级→功能级→领域级，抽象度递增、难度递增、工具支持递减、人工参与递增。重构是在同一抽象层级中转换系统描述。对逆向工程形成的系统修改或重构生成新版本，称为重构工程。

💻 伪代码

// 软件重用类型
水平式重用: 不同应用领域 → 标准函数库
垂直式重用: 共性应用领域 → 区块链

// 逆向工程信息恢复级别(抽象度递增)
实现级: 语法树、符号表
结构级: 程序分量间关系(调用图)
功能级: 功能和程序段关系
领域级: 实体与应用域关系
// 规律: 抽象级别↑ = 难度↑ = 工具支持↓ = 人工参与↑

// 重构(Restructuring): 同一抽象层级中转换
// 重构工程: 逆向工程 + 修改/重构 → 新版本
// 设计恢复: 逆向工程得出的设计，不一定还原到原设计

📌 实际例子

  • 水平式重用：在Web项目中复用标准加密函数库
  • 垂直式重用：多个金融系统共用区块链底层平台
  • 逆向工程：分析旧系统源代码，生成设计文档
  • 重构：将单体架构拆分为微服务(同一抽象层级转换)

✅ 优点

  • 重用大幅提高开发效率
  • 逆向工程帮助理解遗留系统
  • 重构改善系统结构

❌ 缺点

  • 逆向工程不一定能还原到原设计
  • 重构成本可能较高
  • 高级别逆向工程人工参与度大
