网络工程像盖楼：先规划需求，再设计图纸（三层架构），最后施工；关键通道要有备用路线防断。

网络工程三环节：
1. 网络规划 → 需求导向 + 技术/工程可行性
2. 网络设计 → 逻辑设计(结构/选型/IP/路由/冗余/安全)
              + 物理设计(布线/机房/设备选型)
3. 网络实施 → 工程计划 + 设备验收 + 安装调试 + 试运行 + 培训

分层设计三层模型：
接入层 → 面向用户, 带宽+用户管理(地址/认证/计费)
汇聚层 → 访问策略控制, 数据包处理/过滤/寻址 (可选)
核心层 → 高速转发, 可靠骨干传输, 双机冗余+负载均衡

// 冗余设计要点
冗余目的: 避免单点失效
备用路径: 主路径失效时启用(不同时使用!)
负载分担: 并行链路同时分担流量
核心：网络工程分规划、设计、实施三步。网络设计采用三层架构：接入层(面向用户)、汇聚层(策略控制，可选)、核心层(高速可靠骨干)。冗余设计避免单点失效，备用路径在主路径失效时启用，负载分担则通过并行链路同时提高性能。
大型企业网络：接入层交换机连接员工电脑，核心层双机热备份路由器保障骨干可靠 数据中心网络设计中，核心层交换机采用双机冗余热备份+负载均衡，避免单点故障导致业务中断 校园网改造：先做需求调研(规划)，再设计三层拓扑和IP方案(设计)，最后采购设备部署调试(实施)

✅ 优点

❌ 缺点

  • 三层分层设计增加了设备投资和管理复杂度
  • 小型网络不一定需要汇聚层，强行分层反而浪费资源
  • 冗余设计虽提高可靠性，但增加了硬件成本和运维复杂度
