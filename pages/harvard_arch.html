哈佛体系结构（DSP专用）

哈佛结构将程序和数据分开存储于两个独立空间，有两组总线可同时访问，数据吞吐率加倍。

📖 核心解释

哈佛结构将程序存储器和数据存储器分开，各有独立的地址总线和数据总线，可同时对指令和数据进行了访问，数据吞吐率加倍，取指和执行可完全重叠。DSP（数字信号处理器）常采用此结构以实现高速实时信号处理。

💻 伪代码

// 哈佛结构：指令和数据分开存储，有独立总线
CPU
├── 指令总线 <-> 指令存储器(ROM)
└── 数据总线 <-> 数据存储器(RAM)
// 可同时进行取指和执行，数据吞吐率加倍

📌 实际例子

  • DSP（数字信号处理器）采用哈佛结构
  • ARM处理器采用改进型哈佛结构（Modified Harvard）

✅ 优点

  • 取指和执行可完全重叠，速度快
  • 数据吞吐率是冯·诺依曼结构的2倍
  • 适合实时数字信号处理

❌ 缺点

  • 结构较复杂，硬件成本更高
  • 不适用于通用计算，灵活性差
