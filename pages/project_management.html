软件项目管理

项目管理就是管进度(按时做完)、质量(做好)、风险(别出意外)的一套方法。

📖 核心解释

软件项目管理包括进度管理、配置管理、质量管理和风险管理。进度管理6过程：活动定义→排序→资源估计→历时估计→制定计划→控制。WBS以可交付成果为导向逐层分解。任务活动图(甘特图)是项目管理的基础工具。风险管理目标：预防风险并应对已发生风险。Boehm分为风险估计(辨识/分析/排序)和风险控制(计划/处理/监督)；Charette分为分析(辨识/估计/评价)和管理(计划/控制/监督)。

💻 伪代码

// 进度管理6过程
活动定义 → 活动排序 → 活动资源估计
   → 活动历时估计 → 制定进度计划 → 进度控制

// 甘特图(Gantt Chart)
任务A: ████████░░░░  (第1-4周)
任务B: ░░░░████████░  (第3-6周)
任务C: ░░░░░░░░█████  (第5-8周)

// 风险管理两大模型
Boehm: 风险估计(辨识+分析+排序) + 风险控制(计划+处理+监督)
Charette: 风险分析(辨识+估计+评价) + 风险管理(计划+控制+监督)

📌 实际例子

  • WBS分解：开发电商系统→前端模块→登录页面→UI设计任务→分配给小王
  • 甘特图直观展示各任务的时间安排和并行关系
  • 风险管理：识别'核心开发人员离职'风险→分析影响→制定应对计划
  • 进度管理：先确定要做的活动，再排序，估算资源和时间，最后制定计划

✅ 优点

  • 系统化管理降低项目失败风险
  • WBS使任务分配清晰明确
  • 风险管理防患于未然

❌ 缺点

  • 风险管理需要持续投入
  • 进度估计容易不准确
