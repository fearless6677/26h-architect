OSI七层模型与TCP/IP四层模型

OSI七层：应用→表示→会话→传输→网络→数据链路→物理；TCP/IP四层：应用→传输→网际→硬件

⑦ 应用层直接为终端用户服务，提供应用接口（HTTP/DNS）
⑥ 表示层数据编码转换、加密压缩，使不同终端可互通信
⑤ 会话层建立/维护/终止会话，安全验证，同步
④ 传输层端到端传输，TCP保证无差错按序，服务访问点=端口
③ 网络层源→目的传输，路由选择，虚电路/数据报（IP/ICMP）
② 数据链路层点到点帧传输，差错控制，流量控制（MAC/LLC）
① 物理层比特流传输，机械/电气/功能/规程特性

// OSI七层模型（从上到下）
OSI = {
  应用层:   "直接为终端用户服务, 提供应用接口",
  表示层:   "数据编码转换, 使不同终端可互通信",
  会话层:   "建立维护会话, 安全验证, 同步",
  传输层:   "端到端传输, TCP保证无差错按序传输, 端口",
  网络层:   "源到目的传输, 路由选择, 虚电路/数据报",
  数据链路层: "点到点帧传输, 差错控制, 流量控制",
  物理层:   "比特流传输, 机械/电气/功能/规程特性"
}

// TCP/IP四层: 应用层 → 传输层 → 网际层 → 硬件层
// TCP/IP把OSI上面三层(应用+表示+会话)合并为"应用层"

// 数据封装过程（发送方自上而下）
function encapsulate(data) {
  data = 应用层.addHeader(data, "HTTP头")     // 报文
  data = 表示层.encode(data)                   // 编码/加密
  data = 会话层.addSession(data)               // 会话标识
  data = 传输层.addHeader(data, "TCP头,端口80") // 段(Segment)
  data = 网络层.addHeader(data, "IP头,路由")   // 包(Packet)
  data = 数据链路层.addHeader(data, "帧头MAC") // 帧(Frame)
  data = 物理层.toBits(data)                   // 比特流(Bits)
  return data  // → 发送到物理介质
}
核心：OSI/RM七层模型从上到下为：应用层（处理网络应用）、表示层（管理数据表示方式）、会话层（建立维护会话）、传输层（端到端传输，服务访问点为端口）、网络层（源到目的节点传输）、数据链路层（点到点帧传输）、物理层（传输比特流）。TCP/IP四层模型将上面三层合并为应用层。
🌐 HTTP/FTP/SMTP → 应用层 🔒 SSL/TLS → 表示层 🔌 TCP/UDP → 传输层（端口） 📍 IP/ICMP → 网络层 🔗 MAC/交换机 → 数据链路层 ⚡ 光纤/网线 → 物理层

✅ 优点

  • 分层模型清晰定义各层职责
  • 层间解耦便于实现和维护
  • 标准化程度高，利于互操作

❌ 缺点（OSI的不足）

  • 层次过多，实现复杂效率低
  • 会话层/表示层实际功能常被合并
  • 理论模型，实际应用以TCP/IP为主
