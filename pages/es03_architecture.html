// HAL 示例：统一 GPIO 接口，屏蔽不同芯片差异
typedef struct {
    void (*init)(uint32_t pin);
    void (*write)(uint32_t pin, uint8_t val);
    uint8_t (*read)(uint32_t pin);
} HAL_GPIO_t;

// 不同芯片提供不同实现，上层代码无需修改
HAL_GPIO_t stm32_gpio = { .init = stm32_gpio_init, ... };
HAL_GPIO_t nxp_gpio   = { .init = nxp_gpio_init,   ... };

宿主机资源丰富 → 编写代码 → 交叉编译生成目标机指令 → 下载到目标机运行

核心：嵌入式软件采用五层架构，自底向上依次为： (1)硬件层——处理器、存储器、总线、I/O接口等物理设备； (2)抽象层——包含 HAL（硬件抽象层，为上层提供统一的虚拟硬件接口）和 BSP（板级支持包，包含硬件驱动和启动代码）； (3)操作系统层——RTOS、文件系统、GUI 等可配置模块； (4)中间件层——连接两个独立应用的桥梁，如嵌入式数据库、消息中间件等； (5)应用层——各类应用软件。 开发采用宿主机-目标机模式，通过串口/网络/JTAG连接，使用交叉编译器。
🧩 HAL：统一GPIO接口屏蔽芯片差异 🔌 BSP：为ARM Cortex-M4编写启动代码和驱动 🗄️ 中间件：嵌入式SQLite数据库

✅ 优点

  • 层次清晰、职责分明
  • 软硬件解耦——上层不依赖具体芯片
  • 可移植性好——换芯片只需改 HAL/BSP
  • 便于分工协作和独立调试

❌ 缺点

  • 层间调用开销，实时性受影响
  • HAL 设计需兼顾多平台，复杂度高
  • BSP 需针对每款板子单独开发
  • 层数过多降低开发效率
