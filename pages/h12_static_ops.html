静态演化的原子操作（可维护性与可靠性）

静态演化原子操作分为与可维护性相关和与可靠性相关两大类

📖 核心解释

与可维护性相关的操作：AMD(添加模块依赖)、RMD(移除模块依赖)、AMI(添加模块接口)、RMI(移除模块接口)、AM(添加模块)、RM(移除模块)、SM(拆分模块)、AGM(聚合模块)。与可靠性相关的操作：AMS/RMS(消息增删)、AO/RO(对象增删)、AF/RF/CF(片段增删改)、AU/RU(用例增删)、AA/RA(参与者增删)。

💻 伪代码

可维护性相关:
  AMD - Add Module Dependence
  RMD - Remove Module Dependence
  AMI - Add Module Interface
  RMI - Remove Module Interface
  AM  - Add Module
  RM  - Remove Module
  SM  - Split Module
  AGM - Aggregate Modules

可靠性相关:
  AMS/RMS, AO/RO, AF/RF/CF, AU/RU, AA/RA

📌 实际例子

  • 拆分大模块为两个小模块 → SM(Split Module)
  • 合并多个相似模块 → AGM(Aggregate Modules)
  • 添加新的API接口 → AMI(Add Module Interface)
  • 增加新的参与者角色 → AA(Add Actor)

✅ 优点

  • 操作粒度细,便于精确控制
  • 分类覆盖可维护性和可靠性

❌ 缺点

  • 操作种类多达18+种,记忆负担大
  • 需结合具体场景选择合适操作
