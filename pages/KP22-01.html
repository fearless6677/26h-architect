// ① 采集 —— Kafka 消息队列实时接入
val stream = KafkaConsumer(topic = "raw-data")

// ② 清洗 —— Spark 去噪、补全、格式化
val cleaned = stream
    .dedup()                        // 去重
    .fillNull()                     // 补全缺失
    .normalize()                    // 统一格式

// ③ 统计 —— MapReduce 聚合 / OLAP
cleaned.groupBy(_.category).count()

// ④ 挖掘 —— 找相关性，发现新知识
val rules = Apriori(cleaned, minSupport = 0.3)

核心：大数据利用分4个阶段——采集→清洗→统计→挖掘。 传统数据库面临数据过载问题，难以应对类型多样化的数据。 大数据应用重在发掘数据间的相关性而非因果关系，目的在于发现新知识、洞悉趋势并进行科学决策。

💡 关键洞察：相关性 > 因果性

大数据应用不追问"为什么"，而关注"什么总一起发生"。例：啤酒与尿布一起购买频次高 → 直接调整货架即可，无需研究因果。

❌ 因果性：为什么A导致B？ ✅ 相关性：A和B总一起出现！
