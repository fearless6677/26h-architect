层级越高，处理能力越强，覆盖范围越广

设备与OSI层级对应：
物理层(L1):      集线器(Hub), 中继器(Repeater)
数据链路层(L2):  网桥(Bridge), 交换机(Switch)
网络层(L3):      路由器(Router), 防火墙(Firewall)

// 交换机核心功能
交换机 = 集线 + 中继 + 桥接 + 隔离冲突域
STP(生成树协议) → 解决链路环路
链路聚合协议 → 提升端口带宽+链路可靠性

// 路由协议分类
IGP(内部网关协议) → AS内部路由 (如OSPF, RIP)
EGP(外部网关协议) → AS之间路由 (树型拓扑)
BGP → Internet唯一网关协议 (替代EGP)
核心：网络设备按OSI层级工作：集线器/中继器在物理层(传比特流)，交换机/网桥在数据链路层(传帧)，路由器/防火墙在网络层(传包)。交换机用STP解决环路，用链路聚合提升带宽。路由协议分IGP(自治系统内)和EGP/BGP(自治系统间)。
企业办公网中，交换机连接所有办公电脑组成局域网，路由器连接互联网 数据中心核心层使用双机冗余热备份交换机+负载均衡保证可靠性 Internet骨干网通过BGP协议在不同运营商(AS)之间交换路由信息

✅ 优点

❌ 缺点

  • Hub工作在物理层，所有端口共享带宽，容易产生冲突
  • STP虽然解决环路但会阻塞冗余链路，降低带宽利用率
  • BGP配置复杂，策略多变，对运维人员技术要求高
