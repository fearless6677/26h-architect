局域网网络架构

局域网网络架构分为单核心、双核心、环型和层次型四种类型，各有不同的优缺点和适用场景。

单核心架构：使用单台核心交换设备，所有接入交换机直接连到核心。结构简单，投资节约，适合小型网络。
⚠️ 但核心一旦故障，整个网络瘫痪——单点故障风险大。
双核心架构：采用两台核心三层交换机互相冗余。一台挂了另一台还能顶，提高可靠性。
⚠️ 但投资更高，且核心设备端口密度要求较高。
环型架构（RPR）：多台交换机组成双动态弹性分组环（RPR）。具备自愈保护能力，节省光纤资源，提供多等级QoS服务。
⚠️ 投资较高，路由冗余设计实施难度大。
层次型架构：由三层组成——
🔵 核心层：高速数据转发，是网络的"高速公路"
🟡 汇聚层：互访控制、策略管理
🟢 接入层：终端用户接入
✅ 易扩展、便于分级维护，适用于大中型企业网络。

// 局域网网络架构 —— 四种类型对比

// 1. 单核心架构
class SingleCore {
    core = new Switch("核心交换机")   // 只有一台！
    risk = "单点故障"               // 核心挂 = 全网瘫
}

// 2. 双核心架构
class DualCore {
    core1 = new L3Switch("核心1")    // 主
    core2 = new L3Switch("核心2")    // 备/负载均衡
    reliability = "高"             // 冗余备份
}

// 3. 环型架构（RPR）
class RingArch {
    ring = new RPR()               // 双动态弹性分组环
    features = ["自愈保护", "节省光纤", "多等级QoS"]
}

// 4. 层次型架构
class HierarchicalArch {
    core = "核心层"     // 高速数据转发
    distribution = "汇聚层" // 互访控制
    access = "接入层"  // 用户接入
    benefit = "易扩展 + 分级维护"
}
核心：局域网有四种架构——单核心（简单但单点故障）、双核心（冗余但费钱）、环型（RPR自愈保护）、层次型（三层分级，大中型首选）。 选哪个看场景：小型选单核心，大中型选层次型。
🏢 小型网络 → 单核心 🏗️ 大中型企业 → 层次型 🔄 RPR自愈环 → 城域网 🔒 高可靠需求 → 双核心

✅ 优点

  • 层次型架构易扩展、分级维护方便
  • RPR环型架构具备自愈保护能力
  • 单核心架构结构简单、投资节约

❌ 缺点

  • 单核心架构存在核心单点故障风险
  • 环型架构投资较高且路由冗余设计实施难度大
  • 双核心架构核心端口密度要求较高
