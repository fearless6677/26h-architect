TCP与UDP协议对比

TCP可靠面向连接适合小数据量；UDP不可靠无连接适合大数据量快传输；两者共有端口寻址

🛡️ TCP — 可靠传输

三次握手→连接建立
发送数据段→ACK确认
差错校验→重传机制
流量控制→滑动窗口
拥塞控制→慢启动
四次挥手→连接断开
传输速度：较慢

⚡ UDP — 快速传输

直接发送→无需连接
数据报→不等ACK
无校验重传→丢失不管
无流量控制→全速发送
无拥塞控制→最大带宽
发送完毕→无需断开
传输速度：极快
🤝 共同点：都有 端口寻址 能力 
// TCP vs UDP —— 传输层两大核心协议

// ═══ TCP：可靠、面向连接 ═══
TCP 特性：
  ├─ 三次握手 建立连接（SYN → SYN-ACK → ACK）
  ├─ 差错校验 + 超时重传 → 数据完整性
  ├─ 流量控制（滑动窗口）→ 不淹没接收方
  ├─ 拥塞控制（慢启动/拥塞避免）→ 不压垮网络
  └─ 四次挥手 断开连接
  适用：数据量小，可靠性要求高
  场景：HTTP / FTP / SSH / 文件传输

// ═══ UDP：不可靠、无连接 ═══
UDP 特性：
  ├─ 无连接 → 直接发送，无需握手
  ├─ 无重传 → 丢包不管
  ├─ 无流控/拥塞控制 → 全速发送
  └─ 开销极小（8字节头 vs TCP 20字节头）
  适用：数据量大，可靠性要求不高，要求速度快
  场景：视频直播 / DNS查询 / 在线游戏 / VoIP

// ═══ 共同点 ═══
共同：都有端口寻址能力（TCP/UDP端口 0~65535）
核心：TCP是可靠的、面向连接的协议，具有差错校验重传、流量控制、拥塞控制，适用于数据量小且可靠性要求高的场景。 UDP是不可靠的、无连接的协议，适用于数据量大、对可靠性要求不高但要求速度快的场景。两者共同点是都有端口寻址能力。
📁 文件传输(HTTP/FTP)使用TCP保证数据完整性 📺 视频直播/DNS查询使用UDP追求速度

✅ 优点

  • TCP保证数据完整有序到达
  • UDP传输效率高、开销小
  • TCP有流量控制和拥塞控制
  • UDP适合实时应用(直播/游戏)

❌ 缺点

  • TCP开销大、速度慢
  • UDP不保证送达和顺序
  • TCP头部20字节，开销大
  • UDP无拥塞控制，可能丢包

📝 考试关键词

TCP UDP 面向连接 无连接 可靠 不可靠 端口寻址 流量控制 拥塞控制

🔗 关联知识

  • OSI七层模型 = TCP/UDP位于传输层（第4层）
  • 应用层协议 = HTTP/FTP用TCP，DNS用UDP
  • 三次握手/四次挥手 = TCP连接建立与断开过程
  • 端口号 = 0~65535，区分不同应用进程
