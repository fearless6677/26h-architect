效用树（Utility Tree）：将抽象的"效用"分解为四大质量属性，再细化为具体可测量的场景。场景按(优先级, 难度)排序，指导评估聚焦最重要的架构决策。 
// ATAM —— Architecture Tradeoff Analysis Method
// 架构权衡分析法：开发前评估多质量属性

class ATAM {

  // ========== 阶段1: 演示 Presentation =========
  phase1_presentation() {
    introduce("ATAM评估过程")
    introduce("业务驱动因素")      // 功能/利益相关方/业务目标/限制
    introduce("体系结构")          // 可用性及质量要求
  }

  // ========== 阶段2: 调查和分析 =========
  phase2_investigation() {
    architecture_methods = determine_methods()   // 确定架构方法
    utility_tree = build_utility_tree()          // 生成质量属性效用树
    for method in architecture_methods:
      investigate(method)                        // 调查架构方法
      questions = create_analysis_questions()    // 创建分析问题
      answers   = analyze_answers(questions)     // 分析答案
      identify(answers)                          // 找出↓
        风险(Risks)        // 可能导致问题的决策
        非风险(Non-Risks)  // 好的、安全的决策
        敏感点(Sensitivity) // 对某属性影响大的参数
        权衡点(Tradeoff)   // 影响多个属性的决策
  }

  // ========== 阶段3: 测试 =========
  phase3_testing() {
    brainstorm()                              // 头脑风暴
    scenarios = prioritize_scenarios()           // 优先排序场景
    analyze_architecture_methods(scenarios)     // 再次分析
  }

  // ========== 阶段4: 报告 =========
  phase4_reporting() {
    present_all_findings(利益相关者)            // 提供所有信息
  }
}

// 核心概念: 以属性作为架构评估的核心
// 关键工具: 效用树 = 效用 → 质量属性 → 具体场景
// 评估技术: 场景技术 + 启发式分析方法
// 知识库:   有基于属性的体系模型, 可复用

阶段1演示 Presentation

  • a. 介绍ATAM评估过程 — 让利益相关者了解方法
  • b. 介绍业务驱动因素 — 功能需求、利益相关方、业务目标、约束限制
  • c. 介绍要评估的体系结构 — 可用性及质量要求概述

阶段2调查和分析

  • a. 确定架构方法 — 识别当前采用的架构策略
  • b. 生成质量属性效用树 — 确定最重要的质量属性并排序
  • c. 分析体系结构方法 — 循环执行：
    • 调查架构方法 → 创建分析问题 → 分析答案
    • 输出：风险 / 非风险 / 敏感点 / 权衡点

阶段3测试

  • a. 头脑风暴和优先场景 — 利益相关者提出新场景并排序
  • b. 分析架构方法 — 对头脑风暴产生的新场景进行分析

阶段4报告

  • 提供评估期间所有信息给利益相关者 — 包括发现的风险、非风险、敏感点、权衡点
核心：ATAM全称Architecture Tradeoff Analysis Method， 是在系统开发前针对性能、可用性、安全性、可修改性等质量属性进行评价和折中的架构评估方法。 以属性作为核心概念，使用效用树（Utility Tree）对质量属性分类和优先级排序。 评估采用场景技术 + 启发式分析方法，知识库基于属性模型可复用。

💡 效用树示例

  • 效用 → 性能 → (a) 正常负载下0.5秒内响应用户交易请求
  • 效用 → 安全性 → (b) 信用卡支付保证99.999%安全性
  • 效用 → 可用性 → (d) 网络失效后2分钟内发现错误并启用备用系统
  • 效用 → 可修改性 → (h) 在30人月内添加新事务处理中间件
🎯 多质量属性折中 🌳 效用树排序 🔍 风险/敏感点/权衡点 📊 场景+启发式分析 📖 知识库可复用

✅ 优点

  • 系统化地处理多个质量属性之间的权衡
  • 效用树工具使质量属性优先级清晰
  • 知识库可复用，有基于属性的体系模型

❌ 局限性

  • 评估过程复杂，需要大量时间和人力
  • 依赖利益相关者的参与程度
  • 需要评估者具备丰富的架构经验

📝 考试关键词

ATAM Architecture Tradeoff Analysis Method 效用树 Utility Tree 演示 调查和分析 测试 报告 性能 可用性 安全性 可修改性 折中 风险 非风险 敏感点 权衡点
