解释性、交互式、迭代的过程：信息提取 → 数据库构建 → 视图融合，加上可视化交互与模式定义识别两大主要活动。

// Software Architecture Reconstruction —— 解释性·交互式·迭代

function reconstruct(system) {
    // ① 信息提取：分析现有设计与实现工件，构造系统模型
    let model = extract(system.design + system.impl);

    // ② 数据库构建：将视图元素与关系转为标准存储格式
    let db = buildDB(model.elements, model.relations);

    // ③ 视图融合：理顺、加强、建立元素之间的连接
    let fused = fuseViews(db);  // reconcile + augment + connect

    // ④ 可视化与交互：用户操作架构元素
    visualize(fused);  // 提供 UI 让用户浏览/操作

    // ⑤ 模式定义与识别：识别架构模式
    let patterns = recognize(fused, definedPatterns);

    // 🔄 迭代：重复以上过程直到架构清晰
    while (!architecture.clear) {
        fused = fuseViews(rebuild(patterns));
    }
    return architecture;
}

// 关键：①→②→③ 顺序执行；④⑤ 是贯穿全程的主要活动
核心理解：软件架构重构 = 5 个活动。前三个 顺序执行（提取 → 构建 → 融合），后两个 贯穿全程（可视化交互 + 模式定义识别）。整个过程是 解释性的（需要人为判断）、交互式的（用户参与操作）、迭代的（反复精炼）。
① extraction → 分析设计/实现工件 ② DB construction → 标准化存储 ③ view fusion → 理顺·加强·连接 ④ visualization → 用户交互机制 ⑤ pattern → 架构模式识别
