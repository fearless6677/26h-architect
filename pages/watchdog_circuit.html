看门狗电路（Watchdog）

看门狗=定时心跳检测器，程序正常就定期喂狗续命，超时不喂就叫（重启），专治程序跑飞和死锁。

// 看门狗工作流程
timer = 0
while(程序正常运行) {
  timer++
  if(timer >= TIMEOUT) {
    狗叫() // 触发中断或复位MCU
  }
}
// MCU定期喂狗
feed_watchdog() { timer = 0 } // 重置计数器
核心：看门狗是嵌入式系统必备的系统恢复机制，防止程序跑飞或死锁。原理：MCU正常运行时定期喂狗（重置计数器），若程序异常导致超时未喂狗，计数器溢出触发狗叫模块，发出中断或复位信号重启MCU。组成：输入端→寄存器（基本设置）→计数器（计时）→狗叫模块（决定中断还是复位）。
工业PLC控制器：程序死锁时看门狗自动重启，避免产线停摆 车载ECU：程序异常时看门狗复位，确保刹车系统不失控

✅ 优点

  • 自动恢复机制，无需人工干预
  • 可有效应对程序跑飞、死锁等异常
  • 结构简单，硬件开销小

❌ 缺点

  • 如果程序恰好在错误路径上喂狗，看门狗无法检测
  • 超时重启会丢失当前运行状态和数据
