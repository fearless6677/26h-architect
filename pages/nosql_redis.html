# ===== Redis 常用操作 =====

# 1. string → 文章浏览量计数器
INCR  "article:1001:views"    // +1
GET   "article:1001:views"    // → "523"

# 2. zset → 热门帖子排行榜 Top10
ZADD  "hot:posts"  98  "post:1"
ZADD  "hot:posts"  75  "post:2"
ZREVRANGE "hot:posts" 0 9 WITHSCORES

# 3. set → 用户标签去重
SADD  "user:tags"  "Java" "Redis" "Java"
SMEMBERS "user:tags"            // → {"Java","Redis"}

# 4. hash → 用户信息
HSET  "user:1001" "name" "张三" "age" "28"
HGETALL "user:1001"

# ===== 缓存三大问题 =====

# 穿透 → 查不存在的key → 打穿到数据库
solution: BloomFilter.mightContain(key)
            → false 则直接拒绝，不查DB

# 雪崩 → 大量key同时过期 → 数据库被打爆
solution: expire = base + random(1, 60)  // 加随机值

# 击穿 → 热点key过期 → 大量请求涌入
solution: lock = Redisson.tryLock("hot:key")
            → 拿到锁才查DB并回填缓存

# ===== 持久化 =====
# RDB: 定时快照 → 恢复快，可能丢数据
# AOF: 追加日志 → 恢复慢，数据安全

NoSQL泛指非关系型数据库，不保证ACID特性，但具有易扩展、高性能、灵活数据模型等优势。分为列式存储（Cassandra）、键值对（Redis）、文档型（MongoDB）、图数据库（Neo4j）四大类。


Redis是最常用的键值对缓存数据库，支持string / hash / set / list / zset / pub-sub等多种数据类型，数据存在内存中，读写速度极快。


缓存三大问题：

  • 缓存穿透 — 恶意用户用不存在的ID疯狂查询 → 布隆过滤器拦截
  • 缓存雪崩 — 大促时所有商品缓存同时过期，DB被打爆 → 过期时间加随机值
  • 缓存击穿 — 热点key突然过期，大量请求涌入 → 分布式锁保护

Redis持久化两种方式：

  • RDB快照 — 恢复快、数据安全性低，适合备份场景
  • AOF日志 — 恢复慢、数据安全性高，适合对数据安全要求高的场景

核心：NoSQL = 不用表格结构的数据库家族。Redis 是其中最快的"记忆大师"， 把热门数据存在内存里，秒级响应。缓存三大问题必须会背：穿透→布隆、雪崩→随机过期、击穿→分布式锁。

✅ 优点

  • 高性能（内存读写）
  • 易水平扩展
  • 数据模型灵活
  • 支持丰富数据类型

❌ 缺点

  • 不保证ACID
  • 数据可能丢失（内存存储）
  • 不适合复杂查询
  • 缓存一致性问题
