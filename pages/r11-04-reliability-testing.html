// 可靠性测试流程
1. determine_reliability_target()   // 确定可靠性目标
2. develop_operational_profile()    // 开发运行剖面(使用行为建模)
3. design_test_cases()              // 设计测试用例
   // 反映实际使用，优先测试最重要、最频繁功能
4. execute_test()                   // 测试实施
   // 收集4类时间数据：
   //   失效时间 / 失效间隔时间
   //   分组内失效数 / 分组累积失效数
5. analyze_results()                // 测试结果分析
   // 测试记录 + 测试报告
核心：软件可靠性测试是为了获取可靠性数据、评价可靠性水平。 分为广义(建模+统计+分析+评价)和狭义(按用例在预期环境中测试)两种。 关键步骤：确定目标 → 开发运行剖面(使用行为建模) → 设计测试用例(反映真实使用) → 实施测试(收集4类时间数据) → 分析结果。
💡 运行剖面示例：统计发现80%用户只用登录和查询功能，则测试重点覆盖这些高频功能
💡 失效间隔时间：记录相邻两次失效之间的运行时长，用于拟合失效率模型
📊 广义 vs 狭义 🎯 运行剖面 ⏱️ 4类时间数据 🔄 使用行为建模

✅ 优点

  • 测试结果贴近实际使用场景
  • 可发现真实运行中的失效模式

❌ 缺点

  • 测试周期长，需大量时间收集数据
  • 运行剖面建模难度大
  • 测试无法穷尽所有使用场景

📝 考试关键词

广义 狭义 运行剖面 使用行为建模 反映实际使用 失效时间 失效间隔时间 分组失效数 累积失效数
