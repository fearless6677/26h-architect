// 数字签名 = 公钥加密 + 数字摘要(Hash)

// ===== 签名过程 =====
function sign(file, privateKey) {
    hash = SHA256(file)           // ① 对文件做Hash摘要
    signature = Encrypt(hash, privateKey)  // ② 用私钥加密摘要
    return signature               // ③ 得到数字签名
}

// ===== 验证过程 =====
function verify(file, signature, publicKey) {
    hashA = Decrypt(signature, publicKey)  // ① 公钥解密签名 → 摘要A
    hashB = SHA256(file)                  // ② 对文件重新做Hash → 摘要B
    return hashA == hashB                 // ③ 对比：一致则有效
}

// 💡 优化：先做摘要再签名（大幅提升速度）
// 💡 摘要泄露 ≠ 文件泄露（安全！）

核心：数字签名 = 公钥加密 + 数字摘要(Hash)。 实际应用中先对文件做Hash摘要，再对摘要进行私钥签名，这样大大提升签名速度，且摘要泄露不影响文件保密性。 数字签名必须满足5个条件：可信、不可伪造、不可重用、不可改变、不可抵赖。
📝 电子合同签名 💿 软件发布签名 📧 电子邮件签名(PGP) 🔗 区块链交易签名

✅ 优点

  • 防抵赖、防伪造、防篡改
  • 公钥签名无需仲裁，可多方验证
  • 先摘要再签名大幅提升速度

❌ 注意

  • 对称签名需仲裁人，仅限两方
  • 私钥泄露则签名可被伪造
  • 不加密文件本身（需配合加密）

📝 考试关键词

数字签名 数字摘要 Hash 公钥加密 私钥签名 不可抵赖 不可伪造 不可重用 不可改变
