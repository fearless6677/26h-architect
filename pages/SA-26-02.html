// 论文六段式 —— 总字数 2200~2800

/* ① 摘要 (290~320字) —— 不分段！ */
摘要 = "项目背景" + "技术简介" + "效果概述";
// 注意：绝对不能分段，一段写完

/* ② 项目背景 (400~500字) —— 5W2H展开 */
项目背景 = {
    Why:    "为什么做？解决什么问题？",
    When:   "什么时候？",
    Where:  "在哪个单位/地区？",
    Who:    "谁参与？角色是什么？",
    What:   "做了什么系统？",
    HowMuch:"投入多少？规模多大？",
    How:    "怎么做的？用什么技术？"
};

/* ③ 过渡 (~100字) —— 承上启下 */
过渡 = 承上启下(背景 → 正文);

/* ④ 理论 (400~600字) —— 紧扣题干分论点 */
理论 = 紧扣题干("分论点1", "分论点2", ...);

/* ⑤ 实践 (1000~1200字) —— 最重要！Why+How */
实践 = {
    小标题1: "Why → How",  // 为什么这样做 → 具体怎么做
    小标题2: "Why → How",
    小标题3: "Why → How"
};

/* ⑥ 结尾 (300~400字) —— 呼应论点 */
结尾 = 呼应论点() + 效果() + 收获();
维度含义写作示例
Why为什么要做这个项目原有系统效率低下，手工操作易出错
When项目的时间线2023年3月立项，2024年1月上线
Where项目所在的地点/单位某省某银行 / 某市某医院
Who参与人员和你的角色我担任系统架构设计师，团队12人
What做了什么系统绩效考核平台 / 门诊管理系统
How much投入规模和资金总投资200万，覆盖30个分支机构
How采用的技术路线微服务架构，Spring Cloud + Vue

💡 技巧：项目背景可以跨题复用——同一个项目，换不同的技术侧重点。

核心：论文 = 六段式结构，总字数 2200~2800。 其中实践部分最重要（1000~1200字），采用 Why+How 结构。 项目背景用 5W2H 展开，可以跨题复用。
📝 项目背景用5W2H展开：某省某银行绩效考核平台 📐 实践部分用小标题统领，每段Why+How 🔚 结尾呼应论点，简单带过问题

✅ 优点

  • 结构清晰，便于阅卷老师给分
  • 项目背景可跨题复用，准备效率高

❌ 注意事项

  • 理论部分过长会挤占实践篇幅
  • 过渡部分容易被忽略，导致衔接生硬

📝 考试关键词

六段式 5W2H 摘要不分段 实践部分最重要
