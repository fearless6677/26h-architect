系统调查（初步调查与详细调查）

系统调查分初步调查（规划阶段，了解企业概况）和详细调查（分析阶段，深入了解处理流程）两阶段

💻 代码结构

📋 初步调查

阶段：规划阶段 ｜ 由浅入深第一步
  • ① 初步需求分析
  • ② 企业基本状况
  • ③ 管理方式
  • ④ 基础数据状况
  • ⑤ 现有系统状况
➡️

🔍 详细调查

阶段：分析阶段 ｜ 深入了解处理流程
  • ① 运行环境/状况
  • ② 组织结构/业务流程
  • ③ 系统功能
  • ④ 数据资源与数据流程
  • ⑤ 资源情况/约束条件/薄弱环节

// 初步调查 → 规划阶段 → 看"面"（概况、大体印象）
// 详细调查 → 分析阶段 → 看"线"（流程、数据走向）
核心：系统调查分为初步调查和详细调查两个阶段。初步调查在规划阶段进行，了解企业的组织结构和系统功能等，包括初步需求分析、企业基本状况、管理方式、基础数据管理状况和现有系统状况。详细调查在分析阶段深入了解系统的处理流程，包括现有系统的运行环境和状况、组织结构、业务流程、系统功能、数据资源与数据流程、资源情况、约束条件和薄弱环节等。
📊 初步调查 ≈ 企业体检 🔍 详细调查 ≈ 专家会诊 🏗️ 规划阶段 → 分析阶段 📝 由浅入深、由面到线

✅ 优点

  • 两阶段递进式调查确保由浅入深全面了解系统
  • 为后续需求分析和系统设计奠定基础

❌ 缺点

  • 详细调查工作量大、耗时长

📝 考试关键词

初步调查 详细调查 运行环境 业务流程 数据流程 约束条件
🎯 真题：调查现有系统的运行环境和状况、组织结构、业务流程、系统功能等属于（详细调查）

🔗 关联知识

  • 需求分析 = 详细调查之后进入需求分析，将调查结果转化为结构化的需求文档
  • 系统分析 = 在调查结果基础上建立逻辑模型（数据流图、数据字典）
  • 系统规划步骤 = 初步调查是系统规划的组成部分，为可行性研究提供依据
