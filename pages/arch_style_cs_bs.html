C/S与B/S架构风格

C/S与B/S是调用/返回风格的核心架构，三层B/S为浏览器→Web→数据库

📖 核心解释

调用/返回风格采用分而治之策略。C/S架构：二层为数据库服务器+客户应用；三层(瘦客户端)分为表示层、功能层、数据层。B/S是三层C/S的实现方式，三层为浏览器、Web服务器、数据库服务器。B/S相比C/S的不足：动态页面支持弱、扩展能力差、安全性难控、响应速度不足、数据交互性不强。

💻 伪代码

// C/S与B/S对比
CS_BS = {
  二层CS: {
    组成: "数据库服务器(后台) + 客户应用(前台) + 网络",
    优点: "应用与服务器分布运行",
    缺点: "开发成本高, 客户端复杂, 维护升级困难, 移植困难"
  },
  三层CS: {
    组成: "表示层(GUI) + 功能层(业务逻辑) + 数据层(DBMS)",
    别名: "瘦客户端模式"
  },
  BS: {
    组成: "浏览器 + Web服务器 + 数据库服务器",
    不足: ["动态页面支持弱", "扩展能力差", "安全性难控", "响应速度慢", "数据交互性不强"]
  }
}

📌 实际例子

  • 二层C/S: 传统MIS系统，客户端直接访问数据库
  • 三层C/S: ERP系统，客户端→应用服务器→数据库
  • B/S: 网上购物系统，浏览器→Web服务器→数据库

✅ 优点

  • C/S: 应用与服务器分布运行
  • B/S: 客户端零安装, 跨平台

❌ 缺点

  • 二层C/S: 开发成本高, 维护升级困难
  • B/S: 动态页面弱, 扩展差, 安全性难控, 响应慢
