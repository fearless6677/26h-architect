层次化网络设计与网络安全控制技术

层次化网络设计分为接入层、汇聚层、核心层三层，设计应从接入层开始向上规划；网络安全控制包括防火墙、VPN、访问控制、安全隔离和安全协议。

// ===== 层次化网络设计 ===== 三层模型
设计原则（自底向上!）：

  1. 接入层  → 用户接入网络（交换机端口、VLAN划分）
      ↓ 先搞清楚谁要接入
  2. 汇聚层  → 策略控制（ACL、路由聚合、安全策略）
      ↓ 再定规则
  3. 核心层  → 高速转发（骨干传输、快速交换）
      ↓ 最后选核心设备

// ⚠️ 关键：从接入层开始向上规划，不是先设计核心层！
// ⚠️ 严格控制层次结构，不能随意加入额外连接

// ===== 网络安全控制 =====
安全控制五大技术：

  ① 防火墙 ── 网络间的安全屏障
     ├── 包过滤（网络层/传输层）
     ├── 应用层网关（应用层代理）
     └── 代理服务（转发+检查）

  ② VPN ── 公共网络上建私有隧道
     优点：成本低、接入方便、可扩展性强

  ③ 访问控制 ── 谁能做什么
     ├── DAC - 自主访问控制（资源所有者决定）
     ├── MAC - 强制访问控制（系统强制，敏感度标签）
     ├── RBAC - 基于角色的访问控制
     ├── TBAC - 基于任务的访问控制
     └── OBAC - 基于对象的访问控制

  ④ 安全隔离 ── 隔离保护
     子网隔离 / 物理隔离 / VLAN隔离 / 逻辑隔离

  ⑤ 安全协议 ── 加密传输
     ✅ SSL、HTTPS、SET → 是安全协议
     ❌ FTP → 不是安全协议！（明文传输）

// 🌿 绿色网络 = 精简 + 重用 + 回收
核心：网络设计分三层（接入→汇聚→核心），必须从接入层开始向上规划。 安全控制五件套：防火墙、VPN、访问控制、安全隔离、安全协议。记住 FTP 不是安全协议！
🏢 企业网三层架构 🧱 防火墙三种类型 🔑 DAC/MAC/RBAC访问控制 🔐 SSL/HTTPS/SET安全协议 🌿 绿色网络：精简+重用+回收

✅ 优点

  • 层次化设计降低成本，充分利用模块化设备
  • VPN成本低、接入方便、可扩展性强
  • 绿色网络设计采用精简、重用、回收思路

❌ 缺点/注意

  • 不能在设计中随意加入额外连接，灵活性受限
  • 防火墙体系结构复杂（硬件/软件/嵌入式）
  • 子域间路由冗余设计易形成环路

📝 考试关键词

🔗 关联知识

  • 网络基础（20-1） = 网络协议与分层模型，层次化设计的前置知识
  • 网络存储（20-4） = 存储区域网络，常与网络设计一起出题
  • 防火墙 vs 入侵检测：防火墙是"门"（事前拦截），IDS是"监控"（事后发现）
  • VPN三种类型：内联网VPN、外联网VPN、远程访问VPN
  • 访问控制对比：DAC（自主，灵活） vs MAC（强制，严格） vs RBAC（角色，管理方便）
