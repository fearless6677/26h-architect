UML用例图三种关系（include / extend / 泛化）

UML用例图包含3种关系：include（必须包含）、extend（可选扩展）、泛化（子类继承父用例）。

用例图三种关系示意图
<<include>>
基用例 必须执行 子用例 → 提取公共动作
<<extend>>
扩展用例 可选地增强 基用例 → 基用例本身是完整的
<<泛化>>
子用例 继承 父用例 → 特殊 / 一般关系
include（必须）
extend（可选）

// ────────────────────────────────────────────────
// UML用例图三种关系 —— 用伪代码类比理解
// ────────────────────────────────────────────────

// ① <<include>> 包含关系：基用例【必须】执行子用例
function 课程学习() {
    检查权限();          // ← 必须调用，缺一不可
    进入课程页面();
    开始学习();
}

// ② <<extend>> 扩展关系：扩展用例【可选】增强基用例
function 课程学习() {
    检查权限();
    进入课程页面();
    开始学习();
    if (学费不足) {        // ← 特定条件才触发
        缴纳学费();       // ← 可选扩展，基用例本身完整
    }
}

// ③ <<generalization>> 泛化关系：子用例继承父用例
class 课程注册 {           // ← 父用例（抽象）
    验证身份();
    提交注册();
}
class 网络注册 extends 课程注册 {  // ← 子用例（具体）
    选择在线支付();
}
class 电话注册 extends 课程注册 {  // ← 子用例（具体）
    人工确认信息();
}
核心：UML用例图中用例间有3种重要关系：
① 包含关系（<<include>>）：基用例执行时必须包含子用例的行为，用于提取公共动作；
② 扩展关系（<<extend>>）：扩展用例在特定条件下对基用例进行补充，基用例本身是完整的；
③ 泛化关系（<<generalization>>）：子用例继承父用例的结构和行为，是特殊/一般的关系。
🔑 一句话：包含是必须的，扩展是可选的，泛化是继承。
📚 课程学习 include 检查权限（必须执行） 💰 课程学习 extend 缴纳学费（学费不够时才补缴） 📝 课程注册 泛化 网络注册/电话注册

✅ 优点

  • include：提取公共行为，减少重复建模
  • extend：保持基用例简洁，分离可选逻辑
  • 泛化：抽象共性，支持多态与复用

❌ 易混淆点

  • include 和 extend 箭头方向相反
  • extend 基用例必须独立完整
  • 泛化箭头是空心三角，不是普通箭头
  • include 提取公共，extend 添加可选

📝 考试关键词

包含关系 扩展关系 泛化关系 include extend generalization 用例图 公共行为 基用例完整

🔗 关联知识

  • UML 14种图 = 用例图是需求分析阶段最重要的图
  • 面向对象继承 = 用例图泛化关系在代码层面的对应
  • vs 类图泛化：类图泛化是"子类继承父类"，用例图泛化是"子用例继承父用例"
