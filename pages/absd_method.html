基于架构的软件开发方法(ABSD)

基于架构的软件开发(ABSD)是架构驱动、自顶向下、递归细化的方法

📖 核心解释

ABSD是架构驱动的，由商业、质量和功能需求的组合驱动。采用视图描述架构、用例描述功能需求、质量场景描述质量需求。ABSD是自顶向下、递归细化、迭代的过程。ABSDM模型划分六个子过程：需求→设计→文档化→复审→实现→演化。架构复审必须邀请外部人员参与，目的是标识潜在风险、发现设计缺陷。

💻 伪代码

// ABSD三大基础
ABSD_Basis = {
  1: "功能的分解(需求分解)",
  2: "选择架构风格(实现质量和商业需求)",
  3: "软件模板的使用"
}

// ABSDM六个子过程
ABSDM = ["架构需求", "架构设计", "架构文档化", "架构复审", "架构实现", "架构演化"]

// 架构需求三步
RequirementSteps = {
  1: "需求获取(质量目标+商业目标+开发人员目标)",
  2: "标识构件: 生成类图→类分组→打包成构件",
  3: "架构需求评审"
}

// 架构演化步骤
EvolutionSteps = ["需求变化归类", "演化计划", "构件变动", "更新交互", "组装测试", "技术评审", "新架构"]

📌 实际例子

  • 架构需求获取三方面: 质量目标、系统商业目标、开发人员商业目标
  • 标识构件三步: 生成类图→对类分组→打包成构件
  • 架构文档化输出: 架构规格说明 + 测试质量设计说明书

✅ 优点

  • 降低设计随意性
  • 每步迭代有清晰定义
  • 架构驱动保证质量
  • 支持需求演化
