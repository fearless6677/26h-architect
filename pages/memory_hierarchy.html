存储器层次结构（Memory Hierarchy）

存储器按距CPU远近分为片上缓存、片外缓存、主存、外存4层，速度递减，容量递增。

Level 0: 寄存器       ~1ns     几KB     (片上)
Level 1: L1 Cache     ~1ns     几十KB   (片上缓存)
Level 2: L2 Cache     ~3ns     几百KB   (片上/片外)
Level 3: L3 Cache     ~10ns    几MB     (片外缓存)
Level 4: 主存(内存)   ~100ns   几GB     (DRAM)
Level 5: 外存(磁盘)   ~10ms    几TB     (Flash/Disk)

// 访问速度: L0 > L1 > L2 > L3 > L4 > L5
// 存储容量: L0 < L1 < L2 < L3 < L4 < L5
// 单位成本: L0 > L1 > L2 > L3 > L4 > L5
核心：计算机存储器按与处理器的物理距离分为4层：片上缓存（Cache）→ 片外缓存 → 主存（内存）→ 外存（磁盘）。 访问速度依次降低，容量依次提高，单位成本也依次降低。利用局部性原理，将 frequently accessed 数据放在高速缓存中以提升性能。 按介质可分为SRAM、DRAM、NVRAM、Flash、EPROM、Disk等。
💾 L1/L2 Cache用SRAM（速度快、价格高） 🧠 主存用DRAM（速度较慢、价格低） 💿 SSD用Flash作为外存

✅ 优点

  • 利用局部性原理，以低成本获得高性能
  • 层次清晰，利于硬件优化设计

❌ 缺点

  • 高速存储器成本高、容量有限
  • 层次越多，一致性维护越复杂
