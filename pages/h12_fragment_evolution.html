复合片段演化与约束演化

复合片段演化包括AF/DF/FTC/FCC四类操作；约束演化包括AC/DC两类操作

📖 核心解释

复合片段演化针对控制流：AF(新增复合片段)、DF(删除复合片段)、FTC(改变复合片段类型)、FCC(改变执行条件)。约束演化针对约束信息：AC(添加新约束)、DC(移除约束)。复合片段关注控制流的结构化描述，约束关注架构设计的限制条件。

💻 伪代码

复合片段演化:
  AF  - Add Fragment (增添控制流)
  DF  - Delete Fragment (移除控制流)
  FTC - Fragment Type Change (改变控制流类型)
  FCC - Fragment Condition Change (改变执行条件)

约束演化:
  AC - Add Constraint
  DC - Delete Constraint

📌 实际例子

  • 新增一段条件判断逻辑 → AF
  • 移除过时的重试控制流 → DF
  • 将顺序执行改为并行执行 → FTC
  • 将阈值从100改为200 → FCC
  • 新增安全合规约束 → AC

✅ 优点

  • 覆盖控制流和约束两个维度
  • 演化操作语义清晰

❌ 缺点

  • 需理解UML交互图复合片段语法
  • 约束变更可能影响全局
