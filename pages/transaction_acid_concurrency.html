事务ACID特性与并发控制

事务就像快递——要么全部送达(原子性)，要么全部退回；ACID是事务的四大保险：原子、一致、隔离、持久。

-- 事务ACID：转账操作（原子性保证全做或全不做）
BEGIN TRANSACTION
  UPDATE accounts SET balance = balance - 100 WHERE id = 1;  -- A扣款
  UPDATE accounts SET balance = balance + 100 WHERE id = 2;  -- B入账
COMMIT;  -- 提交（持久化），或 ROLLBACK（回滚，撤销全部操作）

-- 封锁类型对比：
-- X锁(排他锁): 独占读写，其他事务不能操作任何该记录
-- S锁(共享锁): 多事务并发读，但谁也不能写

-- 故障恢复：
-- UNDO: 撤销未完成的事务（回滚到BEGIN之前的状态）
-- REDO: 重做已提交的事务（确保持久性，防止宕机丢数据）
核心：事务是DBMS的基本工作单元，具有ACID四大特性： 原子性(全做或全不做)、一致性(从一个正确状态到另一个正确状态)、 隔离性(事务间互不干扰)、持久性(提交后永久保存)。

并发控制通过封锁机制实现：X封锁(排他锁)允许独占读写； S封锁(共享锁)允许多个事务并发读但不允许写。 当数据库出现故障时，通过UNDO(撤销)和REDO(重做)操作恢复数据。
💸 转账: 扣款+入账 要么都成功要么都回滚 🔐 隔离性: A转账时C不能操作同一账户 🔒 X锁冲突: 两个事务争抢同一条记录 🔓 S锁兼容: 多个事务同时读取同一条记录

✅ 优点

  • 保证数据正确性（ACID四大保险）
  • 支持多用户并发访问（封锁机制协调）
  • 故障后可恢复（UNDO/REDO机制）

❌ 缺点

  • 锁竞争影响性能（排队等待）
  • 死锁需要额外处理（检测+解除）
  • 隔离级别越高，并发度越低
