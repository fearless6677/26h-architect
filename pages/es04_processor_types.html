// 嵌入式处理器五大分类
MPU: 微处理器 → 需外部电路板, 如ARM/PowerPC/MIPS
MCU: 微控制器(单片机) → 片内集成存储+外设, 如STM32/8051
DSP: 数字信号处理器 → 哈佛结构, 适合大数据运算, 如TMS320
GPU: 图形处理器 → 浮点+多核并行, 用于AI深度学习
SoC: 片上系统 → 多IC集成, 含处理器+IP核+存储+软件

// 集成度对比
集成度排序: MPU < MCU < DSP < GPU < SoC

// 存储速度（嵌入式常考）
速度: 寄存器组 > Cache(SRAM) > 内存(DRAM) > Flash > ROM
📖 核心讲解：
嵌入式微处理器分五类：
(1) MPU——需专门电路板，集成度低但可靠性高，如ARM、PowerPC；
(2) MCU（单片机）——将核心存储器和外设封装片内，单片化、体积小、功耗低，如8051、STM32；
(3) DSP——采用哈佛结构，对指令特殊优化，适合大量数据处理，如TMS320系列；
(4) GPU——大幅加强浮点和多核并行计算能力，常用于AI深度学习；
(5) SoC——多个IC组合在一芯片上，含完整硬件系统和嵌入式软件，是最高集成度方案。
🏭 MPU：工业控制主板使用ARM处理器 🚁 MCU：STM32F4用于无人机飞控 🎵 DSP：TMS320C6000用于音频信号处理 📱 SoC：高通骁龙集成CPU+GPU+基带

✅ 优点

  • 各有所长，适用不同场景
  • MCU单片化体积小、功耗低、成本低，适合嵌入式场景
  • SoC集成度最高，整体性能最优
  • DSP对数字信号处理效率远超通用CPU
  • GPU大规模并行加速AI训练/推理

❌ 局限

  • MPU集成度低，需外围电路设计，开发周期长
  • MCU运算能力有限，不适合复杂算法
  • DSP功耗较大，不适合电池供电场景
  • SoC设计成本高，流片一次数百万
