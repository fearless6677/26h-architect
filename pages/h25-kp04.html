// Application Architecture Design Process 应用架构设计流程

// ① 输入：Logical DFD + ERD → 概要设计
function createArchitecturalDesign(logicalDFD, ERD) {
    // 受三个因素制约 (constraints)
    const constraints = {
        standards: "architecture standards",   // 架构标准
        goals:     "project goals",           // 项目目标
        tech:      "technical feasibility"     // 技术可行性
    };

    // ② 绘制网络架构DFD
    let networkDFD = drawNetworkArchitectureDFD(logicalDFD);

    // ③ 将数据存储分配到不同处理器
    allocateDataStores(networkDFD, constraints);

    // ④ 分布式数据：分区 or 复制（RDBMS均支持）
    applyDataPartitioning(networkDFD);  // 不同数据 → 不同处理器
    applyDataReplication(networkDFD);   // 同一数据 → 多个处理器

    return networkDFD; // → 详细设计 & 构造 & 实现
}
核心定义：An application architecture specifies the technologies to be used to implement one or more information systems.

设计流程：① 通过逻辑DFD和ERD分配数据和进程 → 创建概要设计 → ② 受架构标准、项目目标、技术可行性制约 → ③ 首先绘制network architecture DFD → ④ 将数据存储分配到不同处理器 → ⑤ 大多数RDBMS支持两种分布式数据形式：data partitioning（分区）和 data replication（复制）。
📘 logical DFD = 逻辑数据流图 🔗 ERD = Entity-Relationship Diagram 📦 data partitioning & replication 🌐 network architecture DFD
