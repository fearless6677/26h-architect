软件可靠性设计技术

通过容错、检错、降复杂度和系统配置提升软件可靠性

📖 核心解释

软件可靠性设计四大技术：(1)容错设计(恢复块/N版本程序/冗余)——自动处理故障；(2)检错设计——代价低但需人工干预，需考虑检测对象、检测延时、实现方法、处理方式四要素；(3)降低复杂度设计——简化结构、缩短代码、优化数据流；(4)系统配置技术——双机热备(热备/互备/双工)和服务器集群(故障自动接管)。

// 容错设计技术
recovery_block():
  try block_A(primary)
  if fail: try block_B(backup)
  if fail: try block_C(fallback)

  result_A = version_A(input)

  result_C = version_C(input)
  output = majority_vote(result_A, result_B, result_C)

redundancy_design():
  if primary_system.fail():
    switch_to(backup_system)  // 不同路径/算法

📌 实际例子

  • 恢复块：主模块崩溃后自动切换到备用模块，如数据库主从切换
  • N版本程序：航天飞控用3个独立团队编写的版本进行多数表决
  • 双机热备模式(Active/Standby)：主服务器宕机后备机自动接管，数据库数据同步写入

✅ 优点

  • 容错设计可自动处理故障，无需人工干预
  • N版本程序可防止单版本设计缺陷
  • 双机热备RTO极短

❌ 缺点

  • 容错和冗余设计成本高
  • N版本程序需独立团队开发，代价大
  • 检错技术不能自动解决故障
  • 冗余系统需保证数据一致性
