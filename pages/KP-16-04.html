// ── ACID事务：银行转账 ──
function transfer(from, to, amount) {
    conn.beginTransaction();          // BEGIN
    try {
        dao.deduct(from, amount);   // 扣款（原子操作1）
        dao.credit(to, amount);     // 入账（原子操作2）
        conn.commit();              // 全部成功 → COMMIT（持久性）
    } catch(e) {
        conn.rollback();            // 任一失败 → ROLLBACK（原子性+一致性）
    }
}
// 隔离性：commit前，其他事务看不到中间状态
核心：数据访问层有5种模式——①在线访问（最常用，每次操作与后台实时交互）；②DAO模式（将底层数据访问与高层业务逻辑分离，含DAO工厂/接口/实现类/DTO）；③DTO模式（跨进程/网络边界传输数据的容器）；④离线数据模式（以数据为中心，操作独立于后台连接和事务）；⑤ORM（对象/关系映射，应用数据与数据库记录互转）。事务处理必须服从ACID原则：原子性（失败则全部失效）、一致性（失败则恢复执行前状态）、隔离性（提交前对其他事务不可见）、持久性（已提交数据状态正确）。工厂模式用于根据数据库类型延迟实例化具体类。
🔌 DAO：JDBC封装UserDao接口及实现类 🔄 ORM：Hibernate将Java对象映射为数据库表记录 🏦 ACID实践：银行转账——扣款和入账必须同时成功或回滚

✅ 优点

  • DAO分离数据访问与业务逻辑，降低耦合
  • DTO支持跨进程/网络数据传输
  • ORM简化数据库操作，提高开发效率
  • ACID保证事务正确性与数据一致性
  • 连接池提高连接复用效率，降低开销

❌ 缺点/注意事项

  • 在线访问每次操作都与后台交互，连接开销大
  • ORM工具可能生成低效SQL，影响性能
  • 连接池需要合理配置参数（最大连接数/超时等）
