// 信息安全威胁分类 —— 被动 vs 主动攻击

class SecurityThreat {
    // 被动攻击：只窃取，不修改信息
    passiveAttack(target) {
        return {
            窃听: "监听通信链路，截取数据",
            信息泄露: "敏感信息被未授权获取",
            业务流分析: "分析通信模式推断内容"
        }[target]
        // ⚠️ 特点：检测难度大，信息未被修改
    }

    // 主动攻击：修改或伪造信息
    activeAttack(target) {
        return {
            非法登录: "未授权访问系统资源",
            假冒身份: "冒充合法用户获取权限",
            拒绝服务: "使系统无法正常服务",
            数据篡改: "修改传输或存储的数据",
            重放攻击: "截获合法数据后重新发送",
            特洛伊木马: "伪装无害程序执行恶意操作",
            陷阱门: "预留后门绕过安全策略"
        }[target]
        // ⚠️ 特点：可检测但损害已造成
    }
}

// 威胁来源6大层面：物理环境→通信链路→网络→操作系统→应用→管理
核心：安全威胁来源包括物理环境、通信链路、网络系统、操作系统、应用系统、管理系统六大层面。 攻击分为被动型（监听、截取，不修改信息只窃取）和主动型（非法登录、假冒、拒绝服务、篡改、重放等，会修改或伪造信息）。 共19种常见威胁类型构成安全检查清单。
🐴 特洛伊木马：伪装无害程序 🚪 陷阱门：系统预留"机关" 🔁 重放攻击：截获后非法重发 🎭 假冒身份：冒充合法用户

✅ 优势

  • 覆盖全面的安全威胁分类体系
  • 区分主被动攻击有助于制定针对性防御策略
  • 19种威胁类型为安全设计提供检查清单

❌ 挑战

  • 被动攻击不修改信息，检测难度大
  • 主动攻击虽可检测但已造成损害
  • 威胁类型繁多，难以全面防护

🔗 关联知识

  • 安全防御策略（kp-21-02） = 针对被动/主动攻击的对应防御方案
  • 加密与认证（kp-21-04） = 防止窃听、篡改、重放的技术手段
  • 陷阱门 vs 特洛伊木马：陷阱门是系统内部预留的后门，木马是伪装的外部恶意程序
  • 被动 vs 主动：被动只看不改（难检测），主动又改又造（可检测但已受损）
