// 交叉开发模型
宿主机(PC) --[串口/网络/JTAG]--> 目标机(嵌入式设备)

// 关键工具链
交叉编译器: 宿主机编译 → 生成目标机可执行代码
交叉链接器: 链接目标机库文件
源代码调试器: 远程调试目标机程序

// 低功耗设计手段
低功耗 = [编译优化, 软硬件协同设计, 算法优化, 中断代替查询, 电源管理]
核心：嵌入式系统最大特点：开发在宿主机（PC）上进行，运行在目标机（嵌入式设备）上，两者通过串口/网络/JTAG连接。由于指令集不同，必须使用交叉编译器在宿主机上生成目标机可执行代码。基本工具：交叉编译器、交叉链接器、源代码调试器。低功耗设计是重要考量，软件层面包括：编译优化、软硬件协同设计、算法优化、用中断代替查询、电源管理。
用PC上ARM交叉编译器编译Linux内核，通过JTAG下载到开发板运行 物联网传感器开发：VS Code写代码→交叉编译→串口烧录到STM32开发板

✅ 优点

  • 利用宿主机强大性能，目标机资源需求极低
  • 交叉编译确保代码与目标硬件指令匹配
  • 低功耗设计延长电池供电设备寿命

❌ 缺点

  • 宿主机和目标机指令不同，调试复杂
  • 需额外交叉工具链和调试设备
  • 串口/网络调试带宽有限

📝 考试关键词
