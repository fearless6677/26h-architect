网络性能指标与非性能指标

网络指标分为性能指标（速率/带宽/吞吐量/时延）和非性能指标（费用/质量/可靠性等）

// 网络指标分类
性能指标 = {
    速率:    数据传输速率 (bps)，如 100Mbps
    带宽:    信道最大数据传输速率，信道的极限能力
    吞吐量:  单位时间实际通过的数据量，受瓶颈影响
    时延:    数据从一端到另一端的总时间
}

非性能指标 = {
    费用:      建网、运维的经济成本
    质量:      用户感知的服务优劣
    标准化:    遵循国际标准（RFC/IEEE）的程度
    可靠性:    抵御故障、持续运行的能力
    可扩展性:  规模扩大的容易程度
    可升级性:  技术迭代的容易程度
    易管理性:  日常管理的便捷程度
    可维护性:  故障诊断与修复的容易程度
}

// 时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延
核心：网络指标分为两大类—— 性能指标从速率、带宽、吞吐量、时延等可量化维度度量网络能力； 非性能指标从费用、质量、标准化、可靠性、可扩展性、可升级性、易管理性、可维护性等综合性维度评价网络质量。
💡 带宽=100Mbps 表示信道最大数据传输速率 ⏱️ 时延=发送时延+传播时延+处理时延+排队时延 📈 吞吐量 ≤ 带宽（受瓶颈链路限制） 🔧 非性能指标决定网络的"好不好用"

✅ 优点

  • 性能指标可量化评估网络能力
  • 非性能指标全面反映网络综合质量

❌ 易混淆点

  • 带宽 ≠ 实际速率（带宽是极限能力）
  • 吞吐量 ≠ 带宽（受瓶颈链路制约）
  • 时延包含4部分，不能只算一种

📝 考试关键词

性能指标 非性能指标 带宽 时延 吞吐量 速率

🔗 关联知识

  • OSI七层模型 → 各层协议的性能直接影响网络指标
  • TCP vs UDP → TCP可靠但时延大，UDP快但不可靠，体现了性能与非性能指标的权衡
  • 网络拓扑 → 影响带宽利用率和时延分布
  • QoS（服务质量） → 综合运用性能与非性能指标保障关键业务
