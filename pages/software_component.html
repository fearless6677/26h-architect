软件构件与三大商用规范（CORBA / J2EE / DNA2000）

软件构件是自包容、可复用的程序集，三大商用规范：CORBA（IDL跨语言）、J2EE（EJB三种Bean）、DNA 2000（DCOM/COM+）。

① CORBA

OMG 规范 · 跨语言分布式
├── ORB（对象请求代理） → 底层通信
├── 公共对象服务（生命周期/事件/持久化…）
└── 公共设施（打印/邮件/文档…）
── 用 IDL 定义接口，跨语言调用

② J2EE

SUN 规范 · 企业级Java
└── EJB 构件标准
    ├── 会话Bean（Session Bean） → 业务逻辑
    ├── 实体Bean（Entity Bean） → 数据库映射
    └── 消息驱动Bean（Message Driven Bean） → 异步消息

③ DNA 2000

Microsoft 规范 · Windows平台
└── DCOM / COM / COM+ 作为标准构件
    ├── COM → 组件对象模型（本地）
    ├── DCOM → 分布式COM（远程）
    └── COM+ → COM+事务/队列/事件服务

核心：软件构件是自包容、可复用的程序集，外部只能通过接口访问它。 两大核心特性：自包容与可重用。 三大商用规范：①CORBA（OMG，IDL跨语言）、②J2EE（SUN，EJB三种Bean）、③DNA 2000（Microsoft，DCOM/COM/COM+）。
🌐 CORBA → 跨语言分布式系统 ☕ J2EE EJB → 企业级Java应用 🪟 COM+ → Windows平台组件开发 🔌 IDL → 接口定义语言

✅ 优点

  • 易扩展，支持并行开发
  • 可重用，降低开发成本
  • 接口标准化，降低耦合

❌ 缺点

  • 需要经验丰富的设计师进行构件划分
  • 快速开发与质量属性之间需妥协
  • 构件质量直接影响软件整体质量

📝 考试关键词

构件 组件 自包容 可复用 CORBA ORB IDL J2EE EJB 会话Bean 实体Bean 消息驱动Bean DCOM COM+

🔗 关联知识

  • 中间件分类 → CORBA/DCOM属于中间件中的ORB/RPC类
  • 面向构件设计 → 构件是比类更大的复用粒度，强调自包容和接口标准化
  • vs 面向对象：对象是代码级复用，构件是二进制级/部署级复用
  • SOA/微服务 → 构件思想的演进，从本地构件到分布式服务
