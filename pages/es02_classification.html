嵌入式系统的分类（实时系统与安全攸关）

按实时性和安全性分为强实时、弱实时和安全攸关三类系统

🔒 安全攸关 (Safety-Critical)：失效 → 人员伤亡/财产损失
飞控、导弹、医疗设备、汽车制动 = 安全攸关  |  智能手表、电子词典 = 非安全攸关 
// 嵌入式系统分类
switch(用途) {
  case "实时系统":
    if(安全性要求严格) return "强实时(Hard Real-Time)";
    else return "弱实时(Weak Real-Time)";
  case "非实时系统":
    return "嵌入式非实时系统";
}

// 安全攸关分类
if(失效后果 == 人员伤亡/财产损失)
  return "安全攸关(Safety-Critical)";
else return "非安全攸关";
核心：按用途分为实时系统和非实时系统。 实时系统又分两种：强实时(Hard Real-Time)——必须在规定时间内完成，否则系统失败； 弱实时(Weak Real-Time)——尽量在规定时间内完成，偶尔超时可接受。 从安全性角度还分为安全攸关系统(Safety-Critical)和非安全攸关系统，前者失效会导致人员伤亡或财产损失。
🛩️ 强实时：飞机飞控、导弹制导 📹 弱实时：视频播放、网络电话 ⌚ 非实时：智能手表、电子词典

✅ 优点

  • 分类清晰，便于设计选型
  • 实时性等级明确，利于确定开发标准
  • 安全攸关维度帮助评估失效风险

❌ 注意

  • 边界模糊：有些系统介于强弱实时之间
  • 安全攸关与非安全攸关并非绝对二分
  • 实际系统可能同时跨越多个分类
