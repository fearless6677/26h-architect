TCP与UDP传输层协议对比

TCP像打电话（先接通再说话，保证听到），UDP像发短信（发出去就行，快但不保收到）。

// TCP连接：三次握手
Client → Server: SYN (seq=x)
Server → Client: SYN+ACK (seq=y, ack=x+1)
Client → Server: ACK (ack=y+1)
// 连接建立，开始传数据

// UDP传输：直接发，不管对方
Client → Server: 数据报(无握手)
// 不保证到达，不保证顺序
核心：TCP是面向连接的可靠协议，提供差错校验、重传、流量控制和拥塞控制，适合小数据量高可靠场景。UDP是无连接的不可靠协议，速度快开销小，适合大数据量、实时性要求高的场景。两者都有端口号寻址能力，这是它们的共同点。
网页浏览(HTTP/HTTPS)、文件传输(FTP)使用TCP，因为要求数据完整无误 视频直播、在线语音通话使用UDP，因为实时性比偶尔丢帧更重要 DNS查询使用UDP(53端口)，因为查询报文小且追求速度；DNS区域传送用TCP

✅ 优点

❌ 缺点

  • TCP：开销大、速度慢，有连接建立和断开的三次握手/四次挥手成本
  • UDP：不可靠，无流量控制和拥塞控制，数据可能丢失、乱序、重复

📝 考试关键词

TCP UDP 面向连接 无连接 可靠 不可靠 端口号 流量控制 拥塞控制
