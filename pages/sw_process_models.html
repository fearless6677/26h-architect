// 选择软件过程模型的决策逻辑
if (需求明确且稳定) {
    选择 = "瀑布模型";
} else if (需求不明确，需要快速确认) {
    if (原型确认后即抛弃) 选择 = "抛弃型原型模型";
    else 选择 = "演化型原型模型";
} else if (项目规模大且风险高) {
    选择 = "螺旋模型"; // 强调风险分析
} else if (需求频繁变化，团队小且协作紧密) {
    if (侧重测试先行) 选择 = "XP极限编程";
    else if (侧重项目管理) 选择 = "Scrum";
    else if (开发人员分类) 选择 = "FDD";
} else if (大型构件化项目，用例驱动) {
    选择 = "RUP统一过程";
}
核心：软件过程模型是软件生命周期中各项工作的规程约束。 常见5种模型：①瀑布模型——因果关系紧密、阶段有里程碑，但需求难确定； ②原型模型——分抛弃型(确认需求后抛弃)和演化型(持续完善)； ③螺旋模型——在原型基础上结合瀑布，强调风险分析； ④敏捷模型（XP/Scrum/FDD/水晶）——迭代开发、快速响应变化； ⑤RUP——用例驱动、以架构为中心、迭代增量。
🏔️ 瀑布适合需求明确的小型项目 🌀 螺旋适合大型高风险项目（航天软件） ⚡ XP核心：测试先行，先写测试再编程 📦 Scrum用Backlog管理，按Sprint迭代 👥 FDD分首席程序员和类程序员

✅ 优点

  • 瀑布模型：流程规范、里程碑清晰
  • 原型模型：有效解决需求不确定性
  • 螺旋模型：强调风险分析，适合大型项目
  • 敏捷模型：快速响应需求变化，高效低风险
  • RUP：用例驱动，迭代增量，架构清晰

❌ 缺点

  • 瀑布模型：需求难以一次确定，变更代价高，各阶段不能并行
  • 原型模型：原型可能被滥用导致质量下降
  • 螺旋模型：开发周期长，成本高
