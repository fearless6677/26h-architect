// DNS查询流程（迭代查询）
客户端 → 本地DNS: 查询 www.example.com
本地DNS → 根DNS: 查 .com 域
根DNS → 本地DNS: 返回 .com DNS服务器地址
本地DNS → .com DNS: 查 example.com
.com DNS → 本地DNS: 返回 example.com DNS地址
本地DNS → example.com DNS: 查 www.example.com
example.com DNS → 本地DNS: 返回IP地址
本地DNS → 客户端: 返回最终IP

// 常见协议端口号速记
FTP: 20(数据) + 21(控制) / HTTP: 80
HTTPS: 443 / DNS: 53 / DHCP: 67,68
TFTP: 69 / SMTP: 25 / Telnet: 23
核心：网络协议是计算机通信的规则约定。常考协议包括：FTP(文件传输,20/21端口)、HTTP/HTTPS(网页传输,80/443端口)、DNS(域名→IP解析)、DHCP(动态分配IP地址)。DNS有两种查询方式：迭代查询和递归查询。HTTPS=HTTP+SSL/TLS加密。
用户访问www.baidu.com时，DNS将域名解析为百度服务器IP地址 企业网络中DHCP服务器自动为新接入的设备分配IP、网关和DNS地址 网银、支付等网站使用HTTPS(443端口)加密传输，防止密码被窃取

✅ 优点

❌ 缺点

  • DNS若被劫持或缓存污染，用户会被导向错误网站
  • FTP使用双连接模式（控制+数据），防火墙配置较复杂
  • DHCP在大型网络中需要合理规划地址池，管理多网段分配较复杂
