阿姆达尔定律(Amdahl)

系统整体提速多少，取决于能加速的那部分占多大比例

📖 核心解释

阿姆达尔定律指出：对某一部件采用更快执行方式所获得的系统性能提升，取决于该方式所占总执行时间的比例。总加速比 = 1/[(1-增强比例) + 增强比例/增强加速比]。加速比取决于两个因素：增强比例（可改进部分占总时间的比例，≤1）和增强加速比（增强后的提速倍数）。

💻 伪代码

// Amdahl定律
增强比例 = f  (能被加速部分占总执行时间的比例)
增强加速比 = k  (加速部分的提速倍数)

新执行时间 = 原时间 × [(1-f) + f/k]
总加速比 = 1 / [(1-f) + f/k]

// 例：60%可加速，加速5倍
加速比 = 1/[(1-0.6) + 0.6/5] = 1/[0.4+0.12] = 1/0.52 ≈ 1.92

📌 实际例子

  • 如果90%的程序可被并行化，加速比为10，则总加速比=1/[(1-0.9)+0.9/10]=1/0.19≈5.26
  • 即使增强比例很大，不可加速部分(1-f)会成为瓶颈——这是Amdahl定律的核心启示
  • 考试常考计算题：给出增强比例和增强加速比，求总加速比

✅ 优点

  • 揭示了系统优化的理论上限
  • 帮助决策应优先优化哪个部件

❌ 缺点

  • 不可加速部分永远是瓶颈
  • 假设增强比例固定不变
