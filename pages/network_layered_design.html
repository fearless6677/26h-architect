  汇聚层: "核心与接入分界, 策略控制/过滤/寻址, 视规模而定",
  核心层: "高速转发, 可靠骨干, 双机冗余热备份, 负载均衡"
}

// 冗余设计
冗余设计 = {
  目的:     "避免单点失效",
  备用路径: "主路径失效时启用, 不承担主路径流量",
  负载分担: "并行链路分担流量, 提高性能"
}
核心：网络分层设计 —— 接入层面向用户连接和访问管理；汇聚层是核心与接入的分界面，完成策略控制和数据处理（视规模而定）；核心层提供高速可靠的骨干传输，需双机冗余热备份。
冗余设计：避免单点失效，备用路径在主路径失效时启用，负载分担通过并行链路提高性能。
🖥️ 企业园区网三层架构 🔧 数据中心核心交换冗余 🔗 HSRP / VRRP 热备份协议 📡 OSPF 多路径负载均衡

✅ 优点

  • 分层设计职责清晰，便于管理和排错
  • 冗余设计提高网络可靠性，避免单点失效
  • 负载分担提升整体网络性能

❌ 缺点

  • 分层增加设备成本和管理开销
  • 冗余设计增加网络复杂度
  • 备用链路平时闲置，利用率低

📝 考试关键词

接入层 汇聚层 核心层 分层设计 冗余设计 单点失效 备用路径 负载分担 双机冗余

🔗 关联知识

  • 网络设备 = 接入层交换机 / 汇聚层三层交换机 / 核心层路由器
  • STP 生成树协议 = 二层冗余防环机制，与冗余设计密切相关
  • HSRP / VRRP = 网关冗余协议，实现核心层双机热备份
  • OSPF / ECMP = 路由层面实现负载分担与多路径冗余
