Architectural Style 架构风格英文核心概念

架构风格英文定义：定义构件和连接器词汇表及组合约束的系统族，典型例子是UNIX shell的管道-过滤器架构。

📖 ENGLISH DEFINITION
An architectural style defines a vocabulary of components and connector types, and a set of constraints on how they can be combined.

// Architectural Style = 一种架构的"定义模板"
// 就像语法：规定了"词汇"和"组合规则"

class ArchitecturalStyle {
    vocabulary = {
        components:   ["filter", "pipe", "repository", ...],
        connectors:   ["pipe", "RPC", "event-bus", ...]
    }
    constraints = [
        "过滤器之间只能单向传递",
        "每个过滤器独立运行",
        "前一个的输出 = 后一个的输入"
    ]
}

// 例子: Pipe-and-Filter Style
const pipeFilter = new ArchitecturalStyle()
pipeFilter.components = ["cat", "grep", "sort", "wc"]
pipeFilter.connectors = ["|"]  // UNIX管道符号

// 经典：cat file | grep "error" | sort | wc -l
# Pipe-and-Filter: 最经典的架构风格实例
$ cat access.log | grep "ERROR" | sort | uniq -c | sort -rn
└─ cat ─┘ └── grep ──┘ └─ sort ┘ └── uniq ──┘ └─ sort ─┘
↑ 每个都是独立的 filter，用 | (pipe/connector) 连接
↑ 完美体现了 architectural style 的定义：
component(cat,grep,sort,uniq) + connector(|) + constraints(单向数据流)
核心：Architectural Style 定义了一个系统族（family of systems）的组织模式。 它规定了component 构件和connector 连接器的词汇表，以及它们的constraints 约束。

管道-过滤器（pipe-and-filter）是最常考的架构风格 —— 每个过滤器（filter）是独立的 component， 管道（pipe）是 connector，数据只能从前往后单向流动（constraints）。
📝 pipe-and-filter architecture = 管道-过滤器架构 🏗️ structural organization = 结构化组织 🧬 semantic models = 语义模型 🖥️ UNIX shell = cat | grep | sort
