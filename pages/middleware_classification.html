// 通信 → 事务 → 数据 → Web → 安全 → 跨平台 → 专用 → 网络
核心：中间件是应用软件与操作系统之间的标准化编程接口和协议，属于可复用基础软件。 常见8类：①通信处理中间件（如MQSeries，保证跨平台通信）； ②事务处理中间件（如Tuxedo，协调处理/负载均衡）； ③数据存取中间件；④Web服务器中间件（如Tomcat、JBOSS）； ⑤安全中间件；⑥跨平台中间件；⑦专用平台中间件；⑧网络中间件。
📨 MQSeries — 企业消息队列通信 🏦 Tuxedo — 银行交易系统事务处理 🐳 Tomcat — Java Web运行时容器 🔐 安全中间件 — 屏蔽OS缺陷

✅ 优点

  • 屏蔽底层差异，简化分布式开发
  • 提供公共服务，提高开发效率
  • 标准化接口，支持异构系统集成

❌ 缺点

  • 增加系统复杂度和部署成本
  • 中间件本身可能成为性能瓶颈

📝 考试关键词

🔗 关联知识

  • 软件构件（software_component） = 中间件是大型可复用构件的典型代表
  • 分布式数据库（distributed_db） = 中间件为分布式DB提供通信与事务支持
  • SOA / 微服务 = 中间件是服务治理的基础设施（ESB → 消息中间件）
  • vs 操作系统：OS管理硬件资源；中间件管理应用间通信与公共服务
