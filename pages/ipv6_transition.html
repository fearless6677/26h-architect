// IP数据报目的地址类型
目的地址 = {
  单播,       // 一对一：发给特定的一台主机
  多播/组播,  // 一对多：发给一组主机
  任播        // 任挑一个：发给最近的任一组
}
核心：IPv6是下一代互联网协议，目的地址有单播、多播/组播、任播三种。 IPv4到IPv6过渡有三种技术：双协议栈（同时运行两种协议）、 隧道技术（IPv6封装在IPv4中传输）、 NAT-PT（地址转换）。
双协议栈同时运行 IPv4 + IPv6

路由器/主机同时拥有IPv4和IPv6地址，两种协议并行工作。终端可直接与IPv4或IPv6节点通信。

隧道技术IPv6 封装在 IPv4 中

两个IPv6孤岛通过IPv4互联网互联时，把IPv6数据报整个作为IPv4数据报的载荷，到达对端再拆封。

NAT-PT地址转换网关

NAT-PT网关做IPv4↔IPv6地址映射与协议转换，让纯IPv6主机能访问纯IPv4服务器，反之亦然。

💡 实例

  • 双协议栈：路由器同时支持IPv4和IPv6，两种地址都能路由
  • 隧道技术：两个IPv6网络通过IPv4互联网互联时，将IPv6包封装在IPv4包中穿越中间网络
🌐 IPv6 128位地址 🎯 单播 Unicast 📢 多播/组播 Multicast 🔄 任播 Anycast 📦 隧道封装 🔀 NAT-PT 地址转换

✅ 优点

  • 双协议栈平滑过渡，不中断现有服务
  • 隧道技术利用现有IPv4基础设施
  • NAT-PT实现地址互通，纯IPv6可访问IPv4

❌ 缺点

  • 双协议栈增加设备复杂度和内存消耗
  • 隧道技术有封装开销，性能下降
  • NAT-PT成为单点瓶颈，转换状态维护成本高
