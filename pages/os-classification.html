操作系统分类与特征

操作系统分为批处理、分时、实时、网络、分布式、嵌入式六类，各有不同的适用场景和特点。

// 操作系统分类
OS = {
  批处理OS:    作业批量提交，无人值守       // 单道/多道
  分时OS:      CPU时间片轮转，多用户交互     // 多路性/独立性/交互性/及时性
  实时OS:      快速响应，高可靠性            // VxWorks, μC/OS-II
  网络OS:      管理网络资源，多用户支持
  分布式OS:    透明性，高性能，网络OS的高级形式
  嵌入式OS:    微型化，可定制，HAL+BSP      // Windows CE, eCos
}
核心：操作系统分为批处理、分时、实时、网络、分布式、嵌入式六类，各有不同的适用场景和特点。
银行交易系统使用实时操作系统（如VxWorks），确保交易在毫秒级内完成响应 手机中的Android/iOS本质上是嵌入式+分时操作系统，支持多任务多用户交互 云计算数据中心使用分布式操作系统，将数千台服务器统一管理调度

✅ 优点

  • 分类明确，不同场景选用不同类型的OS可以最大化系统性能和可靠性
  • 分时系统的多路性和交互性让多用户共享计算资源成为可能
  • 嵌入式OS的HAL和BSP机制使软件易于移植到不同硬件平台

❌ 缺点

  • 实时操作系统开发难度大，对硬件资源要求严格
  • 分布式OS的透明性和一致性实现复杂，调试困难
  • 嵌入式OS高度定制化，移植到新硬件平台成本高
