原型法（快速原型法）

原型法是用系统工具快速搭一个模型给用户看，边看边改，解决需求说不清的问题。

原型法 根据用户初步需求快速建立系统模型进行交流
按功能分 关注原型"做什么" — 界面 or 算法？
水平原型（行为原型）→ 界面/行为 细化需求，不实现功能 — 做UI展示
垂直原型（结构化原型）→ 复杂算法 实现部分功能 — 做底层验证
按结果分 关注原型"用完怎么办" — 扔 or 留？
抛弃式原型（探索式） 解决需求不确定 / 二义性 / 不完整性 / 含糊性
演化式原型 → 逐步演化为最终系统 适合Web项目和易于升级优化的场景
核心记忆： 水平原型 = 看面子（界面展示，后台没数据）； 垂直原型 = 看里子（算法可行，界面不重要）。 抛弃式 = 用完就扔，搞清需求就行； 演化式 = 越改越好，最终变成成品。

原型法的核心流程：快速搭建 → 用户试用 → 反馈修改 → 循环迭代

✅ 最终系统
关键点：原型法的核心是快速和迭代。 不追求一次完美，而是通过反复交流逐步逼近用户真正想要的东西。 特别适合需求说不清、道不明的项目。 
// 原型法 —— 快速搭建模型，边看边改

// ============ 水平原型（行为原型）============
function 水平原型(需求) {
    let 原型 = 快速搭建UI(需求)     // 只做界面，所有按钮都能点
    原型.连接后台(null)             // 但没有真实数据！
    return 原型                        // 给客户看效果、细化需求
}

// ============ 垂直原型（结构化原型）============
function 垂直原型(核心算法) {
    let 原型 = 实现核心功能(核心算法)  // 只做关键算法，验证可行性
    原型.做界面(极简)                // 界面丑没关系，功能跑通就行
    return 原型
}

// ============ 抛弃式原型（探索式）============
function 抛弃式原型(模糊需求) {
    let 原型 = 快速建模(模糊需求)      // 解决需求不确定/二义性/不完整性
    用户.试用并明确需求(原型)          // 客户看完就清楚了
    delete 原型                          // 用完即弃！不是最终代码
    return 用户.明确后的需求
}

// ============ 演化式原型 ============
function 演化式原型(需求) {
    let 系统 = MVP版本(需求)          // 先做核心功能上线
    while (用户.有新需求()) {
        系统 = 系统.迭代优化()        // 逐步演化为最终系统
    }
    return 系统                          // 适合Web项目、易于升级优化
}
📦 生活化实例
水平原型
给客户做一个所有页面都能点但后台没数据的 Demo
垂直原型
实现核心算法模块验证可行性，其他不做
抛弃式
做完 Demo 客户看完需求清楚了就扔掉
演化式
先做核心功能上线，后续不断迭代完善
记忆技巧： 水平原型像样板间——看着漂亮但没水没电； 垂直原型像发动机测试——丑但能跑； 抛弃式像草稿纸——算完就扔； 演化式像养孩子——慢慢长大成人。

✅ 优点

  • 有效解决需求不确定性
  • 用户参与度高，减少返工

❌ 缺点

  • 用户可能误解原型为成品
  • 原型质量可能较差

📝 考试关键词

水平原型 / 行为原型 垂直原型 / 结构化原型 抛弃式 / 探索式 演化式 需求不确定性 / 二义性 / 不完整性 / 含糊性
