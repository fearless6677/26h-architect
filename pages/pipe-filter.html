// 管道-过滤器 —— 像Linux命令管道 |
function process(data) {
    return data
        | filter(x => x.valid)    // 过滤器1: 过滤无效数据
        | map(x => transform(x))  // 过滤器2: 转换格式
        | reduce(sum, x)          // 过滤器3: 聚合统计
}

// 关键：每个过滤器独立！可以单独替换、自由组合
核心：每个过滤器独立运行，不知道前后是谁。 前一个的输出 = 后一个的输入。可以自由组合，像乐高一样。
🖥️ Unix管道 cat|grep|sort 🔧 编译器（词法→语法→优化→代码） 📸 图片处理流水线 🔍 ETL数据处理

✅ 优点

  • 组件可复用、可单独替换
  • 可自由组合、像乐高拼接
  • 支持并行处理

❌ 缺点

  • 不适合交互式应用
  • 难处理复杂状态
  • 过滤器间只能单向传递

📝 考试关键词

数据流 独立过滤器 前输出=后输入 自由组合

🔗 关联知识

  • 责任链模式 = 管道-过滤器的代码实现
  • K8s Sidecar模式 = 管道-过滤器在云原生的应用
  • 队列（Queue） = 管道-过滤器的底层数据结构
  • vs 批处理：管道是流式(一条条)，批处理是整批(攒完再走)
