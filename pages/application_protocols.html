// ===== 关键应用层协议与端口 =====

// FTP — 两条TCP连接，分离控制与数据
FTP  → TCP, 端口20(数据连接) + 端口21(控制连接)
  控制连接: 始终存在，贯穿整个会话
  数据连接: 按需建立，传文件时打开，传完可关闭

// TFTP — 简单文件传输，UDP不可靠
TFTP → UDP, 端口69
  特点: 轻量、无认证，适合局域网启动引导

// HTTP — Web基础协议
HTTP → TCP, 端口80
  明文传输，请求-响应模型

// HTTPS — HTTP + SSL/TLS 加密
HTTPS→ TCP, 端口443
  在HTTP基础上加入SSL/TLS加密层

// DHCP — 动态分配IP
DHCP → UDP
  过程: Discover → Offer → Request → ACK (四步分配IP)

// DNS — 域名解析
DNS  → 正向: 域名→IP地址
         反向: IP地址→域名 (PTR记录)
  查询方式: 迭代查询 / 递归查询
协议 传输层 端口 功能 特点
FTP TCP 20(数据) + 21(控制) 文件传输 两条连接，控制与数据分离
TFTP UDP 69 简单文件传输 轻量不可靠，无认证
HTTP TCP 80 超文本传输 明文，请求-响应模型
HTTPS TCP 443 安全超文本传输 HTTP + SSL/TLS 加密
DHCP UDP 动态分配 IP地址分配 Discover→Offer→Request→ACK
DNS UDP/TCP 53 域名→IP解析 正向/反向(PTR)，迭代/递归查询
核心：FTP使用TCP，建立两条连接（20端口传数据、21端口传控制信息）； HTTP用TCP的80端口；HTTPS加入SSL/TLS加密用443端口； DHCP动态分配IP地址；DNS将域名解析为IP地址，有迭代查询和递归查询两种方式。
📂 FTP上传/下载文件 🌐 浏览器访问网页(HTTP/HTTPS) 🔗 输入域名自动获取IP(DNS) 📶 接入网络自动获取IP(DHCP) 🔄 网络设备固件升级(TFTP)

💡 典型示例

  • FTP两条连接：控制连接(21)始终存在，数据连接(20)按需建立 — 用户登录时用控制连接，传输文件时才建立数据连接
  • DNS迭代查询：本地DNS服务器返回"你应该去问根服务器"的引用，由客户端（或本地DNS）继续追问下一级
  • DNS递归查询：客户端只需向本地DNS发出请求，本地DNS负责一路查到底，最终返回IP地址

✅ 优势

  • FTP双连接分离控制与数据，传输更高效
  • HTTPS加密保证传输安全，防窃听篡改
  • DHCP集中管理IP，提升地址利用率

❌ 注意点

  • FTP明文传输，安全性差(需SFTP替代)
  • HTTP不加密，敏感数据易被窃取
  • TFTP无认证，仅限可信局域网环境

🔗 关联知识

  • TCP vs UDP — FTP/HTTP/HTTPS基于TCP(可靠)，TFTP/DHCP基于UDP(快速)
  • OSI七层模型 — 应用层协议工作在第七层，是用户直接接触的层
  • DHCP与DNS — DHCP分配IP地址，DNS解析域名到IP，两者协同工作
  • SSL/TLS — HTTPS的安全基础，在HTTP和TCP之间加入加密层
