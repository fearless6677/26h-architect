Nonfunctional Requirements 非功能需求四大类型

四种非功能需求类型（英文）：operational（操作环境）、performance（响应时间/容量/可靠性）、security（防破坏/防数据丢失）、cultural and political（国家政策）。

Operational Requirements
操作需求
指定系统必须运行的操作环境及如何随时间变化
📌 operating environment(s)
📈
Performance Requirements
性能需求
关注响应时间、容量、可靠性等量化指标
📌 response time · capacity · reliability
🛡️
Security Requirements
安全需求
保护信息系统免受故意破坏和数据丢失
📌 protect from disruption & data loss
🌍
Cultural & Political Requirements
文化和政治需求
针对系统使用国家的特定要求（法规、语言、习惯）
📌 specific to the countries
// Nonfunctional Requirements — 四大类型速记
class NonfunctionalRequirements {

  // ① 操作需求：系统在什么环境下跑？怎么变化？
  operational  → { operating_env: "Linux/Cloud/On-prem" }

  // ② 性能需求：快不快？扛不扛得住？稳不稳？
  performance  → { response_time: "<200ms",
                    capacity:      "10k QPS",
                    reliability:   "99.99%" }

  // ③ 安全需求：防攻击、防泄漏、防丢失
  security     → { protect: ["disruption", "data_loss"] }

  // ④ 文化和政治需求：到了哪个国家就得守哪国的规矩
  cultural     → { locale_rules: "GDPR / 等保 / 语言" }
}

// 记忆口诀：O-P-S-C  → "Ops, Perf, Sec, Cul"
// 架构第一步：把 NFR 拆成这四类，再逐步细化

核心思路：架构设计的第一步是将非功能需求细化为四类： Operational（操作环境）→ Performance（响应时间/容量/可靠性）→ Security（防破坏/防数据丢失）→ Cultural & Political（国家政策）。 记忆口诀 OPSC，四类分清，考试不丢分。
⚙️ Operational → 操作环境 📈 Performance → 响应时间/容量/可靠性 🛡️ Security → 防破坏 & 数据丢失 🌍 Cultural & Political → 各国特定要求

✅ 优点

  • 四类需求分类清晰，有规律可循
  • 与中文部分质量属性知识相呼应
  • OPSC 口诀便于快速记忆
  • 每类对应明确考点，易拿分

❌ 难点

  • 四种类型的英文名称需准确记忆
  • 容易混淆各类的具体含义
  • 需区分 performance 与 quality attributes 的边界
  • cultural & political 较生僻，容易被忽略

📝 考试关键词

nonfunctional requirements operational requirements performance requirements security requirements cultural and political requirements response time reliability
