数据仓库与商业智能（BI）

数据仓库是企业的数据"档案馆"，专门收集历史数据用于分析决策；OLTP是"前台柜台"做日常交易，OLAP是"幕后分析师"做决策支持。

商业智能三要素：数据仓库 + OLAP + 数据挖掘

📦 数据源POS / ERP / CRM
🏛️ 数据仓库Data Warehouse
📊 OLAP联机分析处理
💡 决策支持
⛏️ 数据挖掘发现隐藏规律
核心流程：多源数据 → ETL抽取/转换/加载 → 数据仓库(归档) → OLAP分析 / 数据挖掘 → 辅助决策
数据挖掘：在没有明确假设的前提下，发现未知但有效、实用的规律。经典案例："买尿布的人也爱买啤酒" 🍺🧒

数据仓库四大特征（考试必背）：

🎯
面向主题
按业务主题组织
如"销售""客户"
🔗
集成
统一编码/命名
消除数据不一致
🔒
非易失（不更新）
数据载入后只读
不做增删改操作
⏳
时变
随时间积累
保留历史快照
记忆口诀："主集非时" → 面向主题、集成、非易失、时变。 数据仓库存储的是历史归档数据，结构简单适合分析，与传统数据库(面向操作)有本质区别。
对比维度OLTP（联机事务处理）OLAP（联机分析处理）
目标日常操作、业务处理分析决策、报表统计
数据特征当前数据、最新状态历史数据、聚集数据
事务特点短小、快速、并发高复杂查询、耗时长
用户量大量用户（一线员工）少量用户（决策者/分析师）
典型场景超市POS收银系统区域经理分析季度报表
关键区别：OLTP像"前台柜台"，处理每一笔交易；OLAP像"幕后分析师"，汇总分析历史数据辅助决策。 
// 商业智能(BI) 三要素
BI = 数据仓库(Data Warehouse) + OLAP + 数据挖掘

// 数据仓库四大特征
class DataWarehouse {
    面向主题    // 按业务主题(销售/客户)组织
    集成        // 统一编码，消除不一致
    非易失      // 数据载入后不更新、不删除
    时变        // 随时间变化，保留历史快照
}

// ETL流程：从数据源到数据仓库
function ETL(sources) {
    for (src of sources) {
        data = Extract(src)      // 抽取: 从POS/ERP/CRM取数据
        data = Transform(data)   // 转换: 统一格式/编码/清洗
        Load(data, warehouse)    // 加载: 写入数据仓库(只追加)
    }
}

// OLAP vs OLAP
OLTP(pos_system)   // → 每秒处理大量小额交易，当前数据
OLAP(warehouse)    // → 复杂查询，分析过去5年销售趋势

// 数据挖掘：无假设发现未知规律
DataMining(warehouse)  // → "买尿布的人也爱买啤酒" 🍺🧒
// 要求：先知(前所未知) + 有效 + 实用
核心：商业智能(BI)是企业对商业数据的搜集、管理和分析过程。 数据仓库是BI的基础，存储历史归档数据；OLAP提供多维分析能力；数据挖掘则在没有明确假设下发现未知信息。
💡 实际应用举例
🏢 企业分析过去5年销售趋势，辅助决策 🛒 超市POS收银系统 → OLTP典型场景 📊 区域经理分析各门店季度报表 → OLAP 🍺 数据挖掘："买尿布的人也爱买啤酒"

✅ 优点

  • 支持复杂分析查询
  • 历史数据集中管理
  • 辅助决策，提升竞争力

❌ 缺点

  • 建设周期长
  • 数据量大，存储成本高
  • 不适合实时操作
