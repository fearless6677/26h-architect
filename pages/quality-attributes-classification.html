// 软件系统质量属性分类 —— 两大维度、十三个属性

质量属性分类:
  开发期质量属性:                  // 关注开发和维护过程
    - 易理解性: 设计被理解的难易程度
    - 可扩展性: 适应新需求增加新功能 (// 也称灵活性)
    - 可重用性: 重用系统或某一部分的难易程度
    - 可测试性: 测试证明满足需求的难易程度
    - 可维护性: 修改缺陷/增加功能的难易程度
    - 可移植性: 从一个环境转移到另一个环境

  运行期质量属性:                  // 关注运行时的表现
    - 性能:   及时提供服务 (// 速度/吞吐量/容量)
    - 安全性: 向合法用户服务, 阻止非授权使用
    - 可伸缩性: 用户数/数据量增加时维持质量
    - 互操作性: 与其他系统交换数据和调用服务
    - 可靠性: 一定时间内持续无故障运行
    - 可用性: 正常工作时间所占比例
    - 鲁棒性: 非正常情况仍运转 (// 健壮性/容错性)

// 记忆口诀：开6运7 = 十三个属性
// 开发期: 理解/扩展/重用/测试/维护/移植
// 运行期: 性能/安全/伸缩/互操作/可靠/可用/鲁棒
核心：软件质量属性分为开发期和运行期两大类。 开发期关注软件在开发维护过程中的特性（6个），运行期关注软件在运行中的表现（7个）。 共13个质量属性，覆盖软件全生命周期。
💡 可扩展性 = 灵活性：适应新需求增加新功能 💡 鲁棒性 = 健壮性 = 容错性：非法操作/故障仍正常运行 💡 可伸缩性：用户数/数据量增加时维持高质量服务

✅ 价值

  • 系统化了质量属性的分类方式
  • 覆盖了软件全生命周期的质量关注点

📝 考试关键词（必考）

开发期质量属性 运行期质量属性 易理解性 可扩展性 可重用性 可测试性 可维护性 可移植性 性能 安全性 可伸缩性 互操作性 可靠性 可用性 鲁棒性 健壮性 容错性
⚠️ 易混淆辨析：
① 可扩展性也称灵活性，指软件因适应新需求或需求变化而增加新功能的能力
② 鲁棒性也称健壮性或容错性，指用户在非法操作或相关软硬件系统发生故障时系统仍能正常运行
③ 可伸缩性指当用户数和数据量增加时，软件系统维持高服务质量的能力
