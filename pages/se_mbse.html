基于模型的系统工程(MBSE)

用模型（而不是文档）来驱动系统工程的全生命周期

📖 核心解释

基于模型的系统工程(MBSE)是建模方法的形式化应用，支持系统需求、分析、设计、验证和确认等活动，持续贯穿所有生命周期阶段。三大支柱是建模语言、建模工具和建模思路。不同阶段产生不同的UML图：需求分析阶段产生需求图、用例图及包图；功能分析与分配阶段产生顺序图、活动图及状态机图；设计综合阶段产生模块定义图、内部块图及参数图。

💻 伪代码

MBSE = {
 定义: "建模方法的形式化应用",
 三大支柱: [建模语言, 建模工具, 建模思路],
 产物: {
   需求分析阶段: [需求图, 用例图, 包图],
   功能分析与分配阶段: [顺序图, 活动图, 状态机图],
   设计综合阶段: [模块定义图, 内部块图, 参数图]
 }
}

📌 实际例子

  • 需求阶段用SysML的用例图描述系统需求
  • 设计阶段用模块定义图(Block Definition Diagram)定义系统架构
  • 三大支柱记忆口诀：语言+工具+思路

✅ 优点

  • 模型可复用
  • 减少文档歧义
  • 支持全生命周期追溯
