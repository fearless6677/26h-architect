文件存储空间管理（3种方法）

文件存储空间管理有3种方法：空闲区表、位示图（0/1表示空闲/占用）、空闲块链；文件存取有顺序和随机两种方式。

💡 核心理解

文件存储空间管理需要跟踪磁盘块的使用情况，常用3种数据结构：①空闲区表（记录连续空闲区域的起止位置）；②位示图（每一位0/1对应一个磁盘块的空闲/占用状态）；③空闲块链（用指针将空闲块链接起来）。文件存取方法有顺序存取和随机存取两种；文件组织方法有连续结构、链接结构和索引结构。

// 文件存储空间管理 3 种方法

// ① 空闲区表 —— 记录连续空闲区域的起始和长度
struct FreeArea {
    startBlock:  起始块号,
    length:      连续块数
}
// 例: [{start:2, len:3}, {start:8, len:2}, ...]

// ② 位示图 —— 每一位(0/1)对应一个磁盘块的状态
//              0=空闲  1=已占用
// 例: 第1字 1 1 0 0 0 1 1 1 ...
//     块0,1占用，块2,3,4空闲，块5,6,7占用...
bitmap[i] = 0  →  第i块空闲
bitmap[i] = 1  →  第i块已占用

// ③ 空闲块链 —— 将空闲块用指针链接起来
struct FreeBlock {
    blockId:   当前块号,
    next:      → 下一个空闲块
}
// 链: Block2 → Block3 → Block4 → NULL

// ── 文件存取方法 ──
顺序存取 → 按顺序依次读写 (seek + read 循环)
随机存取 → 直接定位到任意位置读写 (lseek + read)
核心：操作系统需要跟踪磁盘上每个块的空闲/占用状态。 空闲区表适合连续分配；位示图查找效率高，32位一个字可管理32块；空闲块链灵活但遍历慢。 文件组织有连续结构、链接结构、索引结构三种。
💾 位示图：32位/字，每bit管一块 📁 FAT文件系统使用空闲块链 📖 索引结构支持高效随机存取 🔄 顺序存取 = 磁带模式 🎯 随机存取 = 磁盘直接定位

✅ 优点

  • 位示图：查找空闲块效率高，空间利用率好
  • 空闲区表：适合连续分配场景，分配/回收快
  • 索引结构：支持高效的随机存取

❌ 缺点

  • 位示图占用额外存储空间
  • 顺序存取不适合随机访问场景
  • 空闲块链遍历效率低，指针占空间
