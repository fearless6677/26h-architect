CISC与RISC指令集架构

CISC指令复杂功能强（如x86），RISC指令简单效率高（如ARM），两者是处理器的两大阵营。

// CISC vs RISC 对比
// CISC: 指令复杂、长度可变、一条指令可做复杂操作
//   例: x86 MUL [addr], reg  // 一条指令完成内存取值+乘法
// RISC: 指令简单、长度固定、每条指令做简单操作
//   例: LOAD R1, [addr]  // 取数
//        MUL R1, R2      // 乘法
//        STORE [addr], R1 // 存回
// 国产处理器: 龙芯(LoongArch), 飞腾(ARM), 申威(SW-64)
核心：CISC指令复杂功能强（如x86），RISC指令简单效率高（如ARM），两者是处理器的两大阵营。
Intel/AMD的x86处理器采用CISC架构，广泛用于PC和服务器，指令丰富但功耗较高 ARM处理器采用RISC架构，用于手机（如骁龙、天玑芯片），能效比高 国产龙芯采用LoongArch架构，飞腾采用ARM架构，均属于RISC路线

✅ 优点

  • CISC: 单条指令功能强大，完成复杂任务代码量少，兼容性好
  • RISC: 指令格式统一，流水线效率高，功耗低，适合嵌入式和移动设备
  • RISC-V开源指令集，国产自主可控的重要技术路线

❌ 缺点

  • RISC: 完成同样功能可能需要更多条指令，编译器优化要求高
  • CISC: 指令译码复杂，功耗较高，单条指令执行时间可能较长
  • 两种架构各有适用场景，不存在绝对优劣
