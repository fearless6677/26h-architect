// 信息安全体系架构 —— 5层 + WPDRRC + 深度防御

// ① 五层安全架构（自底向上）
class SecurityArchitecture {
    layers = [
        "物理安全",   // 前提：环境/设备/媒体
        "系统安全",   // 基础：网络结构/OS/应用系统
        "网络安全",   // 关键：访问控制/通信保密/入侵检测/扫描/防病毒
        "应用安全",   // 资源共享/信息存储（含数据库容灾）
        "安全管理",   // 体制/管理平台/人员意识
    ]
}

// ② WPDRRC模型 = 6环节 × 3要素
const WPDRRC = {
    phases: ["预警W", "保护P", "检测D", "响应R", "恢复R", "反击C"],
    elements: ["人员", "策略", "技术"]
    // 每个环节都要有人员执行、策略指导、技术支撑
}

// ③ OSI深度防御 —— 3种方式
const DefenseInDepth = [
    "多点防御",          // 多节点部署，单点失守不致命
    "分层防御",          // OSI各层独立设防
    "支撑基础设施",      // PKI(公钥) + 检测响应
]

// ⚠️ 注意：数据库容灾 = 网络安全(信息存储) + 应用安全，层次有交叉！
核心：安全体系5层自底向上——物理是前提，系统为基础，网络是关键，应用和管理在上层保障。 WPDRRC不是线性的，6个环节循环往复，每个环节都需要人员、策略、技术三要素支撑。 OSI深度防御的思路：不要把所有鸡蛋放一个篮子里——多点 + 分层 + 基础设施。
🏢 物理安全：机房门禁、UPS、防火 🔐 网络安全：防火墙、IDS、VPN 🛡️ WPDRRC：企业安全体系设计框架 🏗️ OSI深度防御：PKI基础设施

✅ 优点

  • 覆盖面广，从物理到管理全链路
  • 适合论文和案例分析题
  • 与工程实践紧密结合

❌ 缺点

  • 层次划分存在交叉（如数据库容灾同时属于网络安全和应用安全）
  • 实际部署成本高，需要大量人力物力

📝 典型真题

  • 真题：'数据库容灾属于？'—— 网络安全（信息存储安全）+ 应用安全
  • 论文题：基于WPDRRC模型设计某企业安全体系，需覆盖 6环节 × 3要素
  • 设计题：OSI深度防御的三种方式—— 多点防御、分层防御、支撑基础设施
