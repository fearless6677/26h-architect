// 总线分类 —— 四种维度

// ① 按信息种类
数据总线 (DB)  : CPU ↔ Memory   // 传送数据，双向
地址总线 (AB)  : CPU → Memory   // 指定地址，单向
控制总线 (CB)  : CPU ↔ 各部件   // 传送控制信号，双向

// ② 按连接部件
片内总线       : CPU内部 ALU ↔ 寄存器
系统/板级总线  : CPU ↔ Memory ↔ I/O    // 核心组件
局部总线       : CPU ↔ 少数高速组件     // 如GPU
通信总线       : 主机 ↔ 外设           // UART/SPI/USB

// ③ 按传输方向
单工    →→→    : 只能单向
半双工  →←     : 轮流双向  (同一时刻单方向)
全双工  ↔      : 同时双向

// ④ 按信号类型
并行总线 : 多位同时传输, 距离近, 一致性要求高  // ISA, PCI
串行总线 : 一位一位传, 距离远, 线少成本低     // UART, SPI, I2C, USB
核心：总线是计算机各部件之间的公共信息通道。 掌握四种分类维度——信息种类（数据/地址/控制）、 连接部件（片内/系统/局部/通信）、 传输方向（单工/半双工/全双工）、 信号类型（并行/串行）——是嵌入式考试必考基础。
🧠 片内总线：CPU内部ALU与寄存器间的数据通路 🔗 系统总线：ARM处理器与SDRAM间的连接 ⚡ 并行总线：ISA、PCI总线 📡 串行总线：UART、SPI、I²C、USB

✅ 优势对比

  • 并行总线：多位同时传输，速度快
  • 串行总线：线少成本低，距离可远
  • 分类清晰，便于选型和设计

❌ 注意事项

  • 并行总线：一致性要求高，距离近（线间干扰）
  • 串行总线：逐位传输，单次带宽低
  • 半双工不能同时收发，全双工需要更多线路

📝 考试关键词

数据总线 地址总线 控制总线 单工/半双工/全双工 并行总线距离近 串行总线距离远
