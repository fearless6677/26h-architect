核心区别：宏内核 = 用户服务与内核服务在同一空间，代码耦合高但调用快； 微内核 = 在不同空间，结构清晰、代码量少，但跨空间调用有性能开销。

记忆要点：任务三态 = 执行 / 就绪 / 阻塞。 执行→阻塞（等I/O）；阻塞→就绪（I/O完成）；就绪→执行（被调度）。 阻塞不能直接变执行，必须先回到就绪队列。

静态 = 设计时确定优先级

EDF
最早截止时间优先
截止时间越早 → 优先级越高
LLF
最低松弛度优先

RMS ⭐
单调速率调度
周期越短 → 优先级越高（最优！）
区分口诀：EDF看截止时间（deadline），LLF看松弛度/紧急度（laxity），RMS看周期长短（period）。 RMS被认为是最优的静态调度算法——判断题常考！

// ===== 嵌入式OS调度算法伪代码 =====

// EDF: 最早截止时间优先
function EDF_Schedule(taskQueue) {
    return taskQueue.sort((a, b) =>
        a.deadline - b.deadline  // 截止时间越早，优先级越高
    )[0];
}

// LLF: 最低松弛度优先
function LLF_Schedule(taskQueue) {
    return taskQueue.sort((a, b) =>
        a.laxity - b.laxity      // 松弛度越小 = 越紧急
    )[0];                        // laxity = deadline - 已执行时间
}

// RMS: 单调速率（周期越短优先级越高）
function RMS_Schedule(taskQueue) {
    return taskQueue.sort((a, b) =>
        a.period - b.period      // 周期越短，优先级越高
    )[0];                        // ⭐ 被认为是最优静态调度
}

✅ 考试重点（为什么重要）

  • 宏内核 vs 微内核对比是案例题常考内容
  • 三种实时调度算法(EDF/LLF/RMS)的区分是选择/案例题重点
  • RMS被认为最优，是判断题常考点

❌ 易错点 / 注意事项

  • 宏内核代码耦合度高，稳定性较差
  • 微内核跨空间调用可能有性能开销
  • 调度算法需根据实时性要求合理选择
