// 看门狗工作流程
watchdog_timer = MAX_COUNT;
while(running) {
    do_normal_work();                   // 正常业务逻辑
    watchdog_timer = MAX_COUNT;        // 定期"喂狗" → 重置计数器
    if (watchdog_timer <= 0) {
        bark_module.trigger();         // 超时未喂狗 → 中断或复位MCU
    }
}

// 关键：程序必须在规定时间内反复"喂狗"
// 否则看门狗认为系统异常，触发复位恢复
核心：看门狗(Watchdog)是嵌入式系统必须具备的系统恢复机制，防止程序出错或死锁。 组成：输入端 → 寄存器 → 计数器 → 狗叫模块。 寄存器进行基本设置，计数器计算超时时间，狗叫模块决定超时后发出中断或复位。 程序正常运行时MCU定期'喂狗'（重置计数器），若超时未喂狗则触发狗叫模块，一般是重启MCU，从而恢复系统正常运行。
🏭 工业控制器：超时2秒，每500ms喂狗 🚗 汽车ECU：任务死锁后自动复位 📡 物联网设备：异常时自动重启恢复通信

✅ 优点

  • 自动恢复、无需人工干预
  • 有效防止程序死锁

❌ 缺点

  • 无法定位根本错误原因
  • 频繁复位可能丢失数据

📝 考试关键词

看门狗Watchdog 喂狗 系统恢复 防止死锁 计数器 中断或复位MCU
