# 26小时通关系统架构师 · AI 互动学习系统

## 项目概述

这是一个基于纯 HTML 的系统架构设计师考试学习系统，26小时通关 236 个核心知识点，包含 267 个交互页面，支持思维导图动画、知识卡片、真题考点等互动学习功能。

## 核心特性

### ✅ 已实现功能

- **知识地图**：基于 markmap 的思维导图动画，支持全屏、搜索、折叠展开
- **知识卡片**：267 个 HTML 页面，每个知识点独立页面
- **视频演示**：3 个架构模式动画视频（管道-过滤器、黑板模式、事件驱动）
- **分类体系**：按"小时"组织，26 小时通关结构
- **考点标记**：热点考点（红色）、普通考点（蓝色）权重标记
- **纯前端**：零依赖后端，纯 HTML + CSS + JavaScript，可离线使用

### 🚧 规划中功能

- **学习进度系统**：跟踪每个知识点的学习状态
- **反馈系统**：用户笔记、问题反馈、知识点评价
- **AI 互动问答**：基于知识图谱的智能问答（需要 AI 辅助）
- **真题练习**：历年真题在线练习和解析
- **学习路径推荐**：根据用户基础推荐学习顺序

## AI 互动学习说明

### 当前可用的 AI 互动方式

1. **知识图谱探索**
   - 在主页的思维导图（markmap）中点击任意节点
   - AI 可以基于当前节点的知识点进行扩展讲解

2. **知识点深入问答**
   - 打开任意知识点页面（如 `pages/17-1.html`）
   - 可以问 AI："详细解释这个页面的内容"、"这个知识点的考试重点是什么"、"给我举个实际案例"

3. **跨知识点关联**
   - 可以问："第 17 章和第 20 章有什么联系？"
   - AI 可以基于知识图谱分析知识点之间的关系

### 使用示例

```
用户：我在看"管道-过滤器架构模式"这个页面，能给我讲讲实际应用场景吗？
AI：[基于页面内容 + 实际案例进行讲解]

用户：这个知识点在考试中通常怎么出题？
AI：[分析历年真题，给出出题规律和答题技巧]

用户：帮我总结一下"软件架构风格"这一章的核心要点
AI：[基于知识图谱生成结构化总结]
```

### 未来 AI 功能规划

- **智能题库**：根据知识点自动生成练习题
- **薄弱点分析**：通过答题记录分析薄弱环节
- **个性化学习路径**：AI 推荐最适合的学习顺序
- **实时问答**：学习过程中随时提问，AI 即时解答

## 技术栈

- **HTML5 + CSS3 + JavaScript**：纯前端，无框架依赖
- **markmap**：思维导图可视化（基于 D3.js）
- **Remotion**：视频动画生成（React 框架）
- **响应式设计**：支持 PC、平板、手机
